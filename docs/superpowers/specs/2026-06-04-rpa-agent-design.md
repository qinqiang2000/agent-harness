# RPA 自动化 Agent 设计文档

**日期**: 2026-06-04  
**状态**: 已批准

---

## 背景

将 agent-harness 的角色从发票云客服助手切换为独立的 RPA 自动化平台助手。用户可以通过对话驱动浏览器执行自动化操作，支持即时执行和固化流程两种模式。

---

## 目标

- 用户描述任务 → agent 直接调用 `agent-browser` 执行浏览器操作
- 用户要求保存固化流程 → 执行后将步骤序列化为 Playwright 脚本存档
- 遇到 `agent-browser` 无法处理的场景 → 降级生成 Playwright 脚本交给用户

## Skill 设计（rpa-agent）

### 触发条件

用户描述需要浏览器操作的任务，或要求执行/保存自动化流程时触发。

### 执行步骤

1. **任务解析**：理解用户意图，拆解为具体可执行步骤（打开 URL、点击元素、输入内容、截图、导出等）
2. **模式判断**：
   - 关键词"保存"、"固化"、"以后每天"、"下次直接跑" → 固化模式
   - 否则 → 即时执行模式
3. **执行**：
   - 优先调用 `agent-browser` 执行
   - `agent-browser` 无法处理 → 生成 Playwright Python 脚本
4. **固化流程存档**：脚本保存至 `data/rpa/flows/<flow_name>.py`
5. **反馈**：每步操作后报告状态；失败时给出排查建议

### 安全规则

- 执行破坏性操作（提交表单、删除数据、支付等）前，必须通过 `AskUserQuestion` 二次确认
- 禁止在用户未明确授权的站点上执行写操作
- 禁止读取或提交包含密码、token 等凭证的表单字段内容
