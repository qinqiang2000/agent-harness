# Agent Harness 维护文档

## 项目概述

基于 FastAPI + Claude Agent SDK 的 AI Agent 服务。通过 Skill 机制扩展能力，支持多租户、多轮对话、SSE 流式响应。

---

## 架构总览

```
客户端请求 POST /api/query
       ↓
┌─────────────────────────────────────────────┐
│  api/routers/agent.py                       │
│  解析请求参数，保存上下文文件                    │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│  AgentService.process_query()               │
│  1. 构建 prompt（prompt_builder.py）          │
│  2. 构建 ClaudeAgentOptions（模型、工具、skill）│
│  3. 获取/创建 ClaudeSDKClient                │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│  Claude SDK 子进程                           │
│  - 加载 agent_cwd/.claude/skills/ 下的 SKILL │
│  - 按 SKILL.md 指令执行任务                   │
│  - 调用工具（Bash、Read、Grep、Skill 等）      │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│  StreamProcessor（api/core/streaming.py）     │
│  解析消息流 → 输出 SSE 事件给客户端            │
└─────────────────────────────────────────────┘
```

---

## 请求流程详解

1. 客户端发送 `POST /api/query`，参数包括 `tenant_id`、`prompt`、`skill`、`language`、`session_id`、`context`、`metadata`
2. `prompt_builder.py` 组装 prompt：
   - 如果指定了 skill → `"严格按skill: {skill} 执行任务\n用户请求: {prompt}"`
   - 附加租户ID、语言、上下文文件路径、metadata
3. `AgentService` 构建 `ClaudeAgentOptions`，传入 skills 列表
4. Claude SDK 子进程启动，自动从 `agent_cwd/.claude/skills/{skill-name}/SKILL.md` 加载对应 skill
5. Claude 按 SKILL.md 中的步骤执行，通过工具（Bash、Read 等）完成任务
6. 响应通过 SSE 流式返回

---

## Skill 机制

### Claude 如何调用 Skill

- 请求中 `skill` 字段指定要使用的 skill 名称
- 如果未指定，使用环境变量 `DEFAULT_SKILLS`（逗号分隔）
- SDK 启动时通过 `ClaudeAgentOptions.skills` 传入 skill 列表
- Claude 自动加载对应的 `SKILL.md` 文件作为执行指令

### Skill 选择逻辑

| 场景 | skills 值 |
|------|-----------|
| 请求指定 skill + 新会话 | `[request.skill]` |
| 未指定 skill + 新会话 | `DEFAULT_SKILLS` 环境变量 |
| 恢复已有会话 | `None`（复用上次） |

## Skill 开发指南

### 目录结构

```
agent_cwd/.claude/skills/
├── my-skill/
│   ├── SKILL.md              # 必须，skill 定义文件
│   └── references/           # 可选，辅助资料
│       ├── some-guide.md
│       └── mapping.md
```

### SKILL.md 格式

```markdown
---
name: my-skill
description: >-
  一句话描述功能和适用场景。
  触发词：关键词1、关键词2、关键词3。
---

# Skill 标题

**约束说明（可选）**

## Step 1：步骤名称

具体指令，告诉 Claude 做什么、怎么做。

## Step 2：步骤名称

可以引用辅助文件：读取 [references/some-guide.md](references/some-guide.md) 获取...

## Step 3：输出格式

定义输出格式模板，Claude 会按此格式回复。
```

### 字段说明

| 字段 | 必须 | 说明 |
|------|------|------|
| `name` | 是 | skill 唯一标识，请求时通过此名称调用 |
| `description` | 是 | 功能描述 + 触发词，Claude 用来判断何时使用 |
| Markdown 正文 | 是 | Claude 执行时遵循的步骤指令 |

### 编写技巧

1. **指令越具体越好** — 模糊指令会导致 Claude 自由发挥，明确写出每一步要做什么
2. **定义安全约束** — 在开头声明禁止/允许的操作，防止 Claude 执行危险命令
3. **提供输出模板** — 定义期望的输出格式，确保回复结构一致
4. **引用辅助文件** — 把变化频繁的数据（如服务器映射、命令模板）放到 `references/` 下，方便单独更新
5. **异常处理** — 列出可能的异常场景和对应处理方式

### 添加新 Skill

1. 在 `agent_cwd/.claude/skills/` 下创建新目录
2. 编写 `SKILL.md` 文件
3. 如需辅助资料，放到 `references/` 子目录
4. 无需重启服务，新 session 会自动加载

### 修改现有 Skill

直接编辑 `SKILL.md` 文件即可。修改后新的 session 立即生效（已有 session 不受影响）。

### 删除 Skill

删除对应目录即可。

## 工具权限配置

### 基础工具列表

定义在 `api/services/agent_service.py` 的 `_BASE_ALLOWED_TOOLS`：

```python
_BASE_ALLOWED_TOOLS = [
    "Skill",        # 调用其他 skill
    "Read",         # 读取文件
    "Grep",         # 搜索文件内容
    "Glob",         # 按模式查找文件
    "Bash",         # 执行 bash 命令
    "Write(**/data/issue-diagnosis/instincts/**)",  # 限定写入路径
    "Edit(**/data/issue-diagnosis/instincts/**)",   # 限定编辑路径
    "AskUserQuestion",  # 向用户提问
]
```

### MCP 工具扩展

通过环境变量 `ALLOWED_MCP_TOOLS` 添加额外工具（逗号分隔），如 Elasticsearch、GitLab 等 MCP server 提供的工具。

### 安全限制

`AgentService.__init__()` 自动生成 `.custom-settings.json`，包含 deny 列表：
- 禁止读取 `.env`、`.pem`、`.key` 等敏感文件
- 禁止读写 `settings.json`
- 额外限制来自 `agent_cwd/.claude/settings.json` 的 `permissions.deny`

## SSE 事件类型

| 事件 | 说明 |
|------|------|
| `heartbeat` | 连接状态（connecting/connected/processing） |
| `session_created` | 新会话创建，返回 session_id |
| `assistant_message` | Claude 的文本回复（流式） |
| `tool_use` | Claude 调用工具 |
| `todos_update` | 任务列表更新 |
| `ask_user_question` | Claude 向用户提问 |
| `transfer_human` | 转人工 |
| `result` | 最终结果（含 session_id、耗时、轮次、是否出错） |
| `error` | 错误信息 |

---

## 会话管理

- 新请求不带 `session_id` → 创建新会话
- 带 `session_id` → 恢复已有会话（多轮对话）
- 会话通过 `SDKSessionCache` 缓存，复用 SDK 子进程，避免冷启动
- 空闲会话 TTL 为 3600 秒（1小时），超时自动清理

## 示例：添加一个新 Skill

假设要添加一个「日志查询」skill：

```bash
# 1. 创建目录
mkdir -p agent_cwd/.claude/skills/log-query/references

```

`agent_cwd/.claude/skills/log-query/SKILL.md`：

```markdown
---
name: log-query
description: >-
  查询应用日志，支持按时间范围、关键词、服务名过滤。
  触发词：日志、log、查日志、报错日志。
---

# 日志查询

## Step 1：解析查询条件

从用户请求中提取：
- service_name: 服务名
- time_range: 时间范围
- keyword: 关键词

## Step 2：执行查询

使用 Elasticsearch MCP 工具查询日志：
- 索引模式: {service_name}-*
- 时间过滤: @timestamp 在 time_range 内
- 关键词过滤: message 包含 keyword

## Step 3：输出结果

按时间倒序展示前 20 条日志，格式：
时间 | 级别 | 消息内容
```

```bash
# 3. 调用时指定 skill
curl -X POST /api/query -d '{"tenant_id":"1","prompt":"查一下 order-service 最近1小时的报错日志","skill":"log-query"}'
```

无需重启，下次请求即生效。
