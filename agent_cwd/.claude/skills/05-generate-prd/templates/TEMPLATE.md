> - 规则包内规则以规则 ID 索引为准，不在本文重复描述逻辑，开发直接读取该特性关联对象的 RulePackage

### 3.2 规则包外的约束

> 以下约束不在 RulePackage 中定义，属于接口层/状态机层约束，须在代码层实现：

| 约束类型                 | 约束内容                            | 处理方式                                |
| -------------------- | ------------------------------- | ----------------------------------- |
| 状态机 Guard            | [对象].status 须为 SXX 或 SXX，否则拒绝执行 | HTTP 409，错误码 `[对象大写]_INVALID_STATE` |
| 服务异常保护               | 服务内部异常时，状态保持 SXX 不前进            | HTTP 500，不写入目标状态，不产生脏状态             |

> 💡 常见约束类型：
> - 全量错误收集：所有规则跑完后统一返回，不因第一条失败短路
> - 幂等保护：同一请求重复调用不产生重复写入
> - 并发保护：同一对象同时只允许一个动作执行

### 3.3 执行策略与状态结果

- **规则版本**：由 `ProfileSelectionRules` 命中的 profile 决定，产品侧统一维护，单特性文档不重复定义规则内容。本节 3.1 中的规则信息均来源于该特性关联对象的 RulePackage，仅作快速索引，如有不一致以 RulePackage 为准。
- **按需加载**：VL4 行业专项规则仅在 `SpecialBusinessType` 命中对应场景时加载，不影响标准路径性能
- **Optional 规则**：依赖外部系统数据的规则首版作为弱校验不阻断，当外部数据可用时升级为 Fatal

| 执行结果 | 目标状态 | 说明 |

| 任一 Fatal 规则失败 | SXX（[状态名] / [枚举值]） | 返回字段级错误清单，可修正后重新提交 |
| 仅 Warning 规则失败 | SXX（[状态名]，同通过） | Warning 不阻断，随错误清单一并返回 |
| 服务内部异常 | 保持 SXX（[初始状态]） | 返回 HTTP 500，不产生脏状态 |
