# RPA 自动化执行

## 核心规则

### 模式识别（强制前置）

**在执行任何操作前，必须判断任务模式。**

**固化模式触发词**（出现以下任意词语）：
- "保存"、"固化"、"以后每天"、"下次直接跑"、"自动定时"、"每天"、"每周"、"定期"

**其他情况** → 即时执行模式

---

## 执行流程

### 1. 任务解析

将用户描述拆解为具体可执行步骤：
- 打开哪个 URL
- 点击什么元素（按钮/链接/菜单）
- 输入什么内容（文本/下拉选择）
- 等待什么条件（页面加载/元素出现）
- 导出/截图/获取什么数据

**若信息不足**（URL 不明确、操作目标模糊）→ 使用 `AskUserQuestion` 向用户确认，不得猜测。

### 2. 安全检查（强制）

**以下操作执行前必须通过 `AskUserQuestion` 二次确认，用户明确同意后才能执行：**

- 提交表单（登录除外）
- 删除数据
- 支付/转账
- 发送消息/邮件
- 任何不可逆的写操作

**以下操作无条件禁止：**
- 读取或提交包含密码、token、密钥的字段内容
- 在用户未明确提及的站点上执行写操作

### 3. 执行策略

#### 优先：agent-browser 直接执行

调用 `agent-browser` 工具按步骤操作浏览器，每步完成后输出简短状态。

**执行报告格式：**
```
✅ 步骤1：已打开 https://xxx.com
✅ 步骤2：已点击「登录」按钮
✅ 步骤3：已输入用户名
⏳ 步骤4：等待页面加载...
```

#### 降级：生成 Playwright 脚本

遇到以下情况时降级：
- `agent-browser` 返回错误或无法识别元素
- 操作涉及文件下载/上传
- 用户明确要求生成脚本

生成 Python + Playwright 脚本，包含完整注释，可直接运行。

### 4. 固化流程存档

**固化模式下，执行完成后**：
1. 将操作步骤序列化为 Playwright Python 脚本
2. 询问用户流程名称（若未提供）
3. 保存至 `data/rpa/flows/<flow_name>.py`
4. 告知用户脚本路径和运行方式：`python data/rpa/flows/<flow_name>.py`

**脚本模板：**
```python
"""
流程名称：<name>
创建时间：<date>
描述：<user_description>
运行方式：python <filename>
"""
from playwright.sync_api import sync_playwright

def run():
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=False)
        page = browser.new_page()
        # 步骤从这里开始
        ...
        browser.close()

if __name__ == "__main__":
    run()
```

## 工具使用规范

### agent-browser 调用规则

- 每次只执行一个原子操作，等待结果后再执行下一步
- 遇到超时或元素未找到，最多重试 2 次，仍失败则降级生成脚本
- 截图用于确认操作结果，关键步骤完成后主动截图

### AskUserQuestion 规则

- 任务信息不足时使用，调用后立即停止等待用户回答
- 二次确认破坏性操作时使用，用户明确说"确认"/"是"/"继续"才执行
- **NEVER** 跳过二次确认直接执行破坏性操作

---

## 输出规范

### 执行过程

- 步骤状态逐条输出（✅ 成功 / ❌ 失败 / ⏳ 进行中）
- 失败时输出：失败原因 + 建议排查方式

### 执行完成

```
任务完成。共执行 N 步，耗时约 Xs。
[固化模式] 脚本已保存至 data/rpa/flows/<name>.py
```

### 无法执行时

```
抱歉，这个任务我无法直接执行，原因是：<具体原因>。
建议：<替代方案或用户可手动操作的步骤>
```
