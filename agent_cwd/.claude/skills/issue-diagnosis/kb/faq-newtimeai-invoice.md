# 新时代开票接口异常 FAQ

> 适用场景：日志中出现 `newtimeai`、`blueTicket`、`fullExteriorInvoke`、`open.gateway.newtimeai.com`、`局端` 等关键词时参考本文件。
> 错误码来源：**新时代网关**（非发票云内部错误码）。

---

## 新时代网关返回码速查（附件20）

| code | msg/message | 含义 | 处理方式 |
|---|---|---|---|
| 200 | 成功 | 业务成功 | — |
| 302 | 请您获取验证码重新登录 | 电票平台登录失效 | 重新调用登录接口（无需验证码的省份直接调用登录即可） |
| 2001 | 时间戳类型错误 | timestamp 生成不正确 | 检查 timestamp 格式（精确到毫秒） |
| 2002 | 该链接已失效 | timestamp 失效 | 重新生成 token |
| 2003 | 签名参数错误 | sign 生成不正确 | 检查 sign 计算方式（MD5 大写32位） |
| 2004 | token 失效，请重新生成 | 智云开放平台 token 失效 | 重新调用生成 token 接口 |
| 2005 | token 无效，请检查 token | 当前 token 的 secret 不是最新的 | 重新生成 token |
| 2006 | 暂无此接口的权限 | 未申请开通该接口权限 | 联系相关人员申请开通 |
| 3002 | 当前 API 有效时间已超期 | 调用接口权限到期 | 联系相关人员续期 |
| 9002 | 请求格式不合法 | token、timestamp、sign 为空 | 检查请求头是否完整 |
| 999 | 未获取纳税人信息 | 智云管理端未维护纳税人数电企业信息 | 在智云管理端查询是否维护了纳税人数电企业信息 |
| 999 | 请进行商户授权 | 开放平台未绑定智云账号 | 在开放平台绑定智云账号 |
| 999 | token 获取失败，请联系管理员 | 开放平台账号异常 | 检查开放平台账号是否正确 |
| 10400 | 有可能开具出来发票 | 局端异常，发票可能已开具 | **先调发票采集接口确认是否已开具，再用相同流水号重试开具** |

---

## Q1: 开票返回 code:302 / "请您获取验证码重新登录"

电票平台登录 session 失效。

处理：重新调用登录接口（`loginDppt` 或 `getDpptToken`）。无需验证码的省份直接调用登录即可，无需等待验证码。

---

## Q2: 开票返回 code:10400

局端异常，**发票可能已经开具成功**，不能直接重试（会导致重复开票）。

处理步骤：
1. 先调用发票采集接口，确认该流水号对应的发票是否已在税局开具
2. 若已开具 → 采集回来关联，不再重试
3. 若未开具 → 用**相同流水号**重新调用开票接口

---

## Q3: 开票返回 code:999 / "未获取纳税人信息"

智云管理端未维护该税号的数电企业信息。

处理：在 XSD 管理后台检查该税号是否已维护数电企业信息。

---

## Q4: 开票返回需要人脸认证（返回 ewm 字段）

响应中 `data` 包含 `rzid`、`ewm`、`ewmly` 字段时，说明当前操作触发了人脸认证要求。

```json
{
  "code": 200,
  "data": {
    "rzid": "6eb01d52...",
    "ewm": "qrcode_id=...",
    "ewmly": "swj"
  }
}
```

处理：
1. 解析 `ewm` 字段内容，生成二维码供用户扫码
2. `ewmly` 为 `swj` 时需用税务局 APP 扫码
3. 调用获取人脸二维码认证状态接口（`getFaceState`）轮询 `slzt`：1=未认证，2=成功，3=二维码过期
4. `slzt=2` 后重新发起开票请求

---

## Q5: 开票 msg 包含以下文本 → 局端异常，建议重试

以下 msg 均属于局端（电子税局）临时异常，非发票云问题，建议稍后重试：

- `发送请求出现异常`
- `504 Gateway Time-out`
- `发票正在开具中，请稍后`
- `当前请求异常,请您稍后再试`
- `发送请求异常!`
- `Timeout: timeout`
- `解析返回数据异常`
- `局端内部异常,请稍后再试`
- `读取局端响应错误`
- `格式化输出错误`
- `局端请求错误`
- `访问数据支撑平台返回报文为空或格式有误`
- `发送请求失败`
- `局端未返回数据,请您重试或稍后再试`
- `内部错误。请稍后再试`
- `0002 当前请求失败`

---

## Q6: 开票 msg 等于以下文本 → 需重新登录

- `无效请求，请重新进入开具页面`
- `身份认证成功时间查询失败：请重新登录后继续办理业务！`
- `请您获取验证码重新登录`

处理：重新调用登录接口后再开票。

---

## Q7: 开票返回 "系统检测到您在当前页已成功开具发票，请查询确认！如需继续开票，请重新进入开票页面"

税局检测到重复开票请求。

处理：先查询该流水号是否已有对应发票，若已开具则直接使用，不再重试。参考 Q2 的处理步骤。

---

## Q8: 开票流程链路说明（用于定位卡在哪一步）

新时代开票完整调用链路：

```
生成 token（genToken）
  → 登录电票平台（loginDppt / getDpptToken）
  → [可选] 人脸认证（getFaceImg → getFaceState）
  → 授信额度查询（queryCredit）
  → 数电蓝票开具（blueTicket）
```

日志中出现 `302` 或登录相关报错 → 卡在登录步骤
日志中出现 `ewm` 字段 → 卡在人脸认证步骤
日志中出现 `10400` → 卡在开票步骤，局端异常
