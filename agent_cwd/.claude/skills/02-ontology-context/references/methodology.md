# 本体驱动产品设计方法论

> 提取自《本体驱动产品设计指导手册 V1.4-beta》第一章

## 1.1 两种视角的本质差异与结合点

产品经理和系统工程师看同一个产品的方式不同，它们描述的是同一件事的两个截面。

```
产品经理视角（自上而下）               工程/本体视角（自下而上）
────────────────────────────────────────────────────────
为什么做？（业务场景）
    ↓
做什么？（业务流程/业务活动）
    ↓                        ←→      对象 + 状态 + 动作 + 函数
产品能力是什么？（产品特性）
    ↓
怎么做？（规格说明）
```

**结合原则：产品特性是本体动作的"用户价值聚合"。**
一个产品特性背后对应 1~N 个本体动作。本体给了精确的积木定义，产品视角告诉你为什么要搭这个房子、用户住进去的体验是什么。

## 1.2 产品特性 ↔ 本体动作的映射关系

| PM 概念       | 本体概念                             | 关系                     |
| ----------- | -------------------------------- | ---------------------- |
| 业务场景        | —                                | 解释触发背景，回答"为什么"         |
| 用户操作        | Action（动作）                       | 1个特性 = 1~N 个动作         |
| 业务规则（用户可感知） | Guard + Function（动作关联函数与具体Rules） | 前置条件 + 函数+校验规则的地区具体规则包 |
| 系统反馈        | Effect + State（动作关联副作用/状态）       | 状态迁移 + 属性写回的结果呈现       |
| 数据字段        | Property（动作关联输入/输出）              | 关键传参与返回字段              |
| 异常场景        | 失败状态节点                           | 状态机中的失败分支              |
| 对象联动        | 状态事件触发+LinkType                  | 上游对象状态变更触发下游对象创建       |

## 1.3 特性及用户故事规格说明结构

每份产品特性规格书，上半部分用业务语言说明what，下半部分用本体精度说明how：

```
特性名称（面向业务的通用命名）
│
├── 一、特性概述      ← 本体视角：对象/动作映射 + 用户视角：提供什么能力
├── 二、用户故事
│     ├── 业务场景    ← PM 主笔：谁 / 什么时候 / 为什么需要这个能力（1-2句）
│     ├── 主故事      ← As a / I want / So that 标准三段式
│     ├── 用户旅程    ← Happy Path + Unhappy Path（重要）
│     └── 验收标准    ← 功能 AC / 合规 AC / 性能 AC，供测试侧直接生成测试集
│
├── ─────────────────── 技术实现HOW ────────────────────────────────────
├── 三、校验规则说明  ← 规则包调用 + 规则包外约束 + 执行策略
├── 四、接口规格      ← 每个覆盖动作一个子节（Guard / 输入参数 / 函数链 / 响应）
├── 五、地区差异说明  ← Profile 行为差异（CN vs 全球通用）
└── 六、附录          ← 本体映射 / 函数链 / 文档引用索引 / 待补充清单
```

## 1.4 PRD 文档颗粒度原则

| 层级     | 问题        | 标准                               |
| ------ | --------- | -------------------------------- |
| 对象规格   | 属性要不要全列？  | 只列实际业务场景中具体动作会被读/写的属性            |
| API 规格 | 接口要多细？    | Guard + 请求参数 + 成功/失败两条响应路径 = 够用  |
| 函数规格   | 要写实现代码吗？  | 优先引用本体中已定义函数，未命中本体函数先明确后人工判断是否新增 |
| 集成规格   | 字段映射要完整吗？ | 按实际业务场景涉及到的动作用到的字段必须完整           |
| 事件规格   | 要时序图吗？    | 事件表足够，时序图留给技术设计文档                |
