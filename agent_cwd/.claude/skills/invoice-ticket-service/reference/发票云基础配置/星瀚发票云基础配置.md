# <font style="color:rgb(0, 0, 0);">二、发票云授权配置</font>
**<font style="color:#DF2A3F;">是否必需：是</font>**

<font style="color:#117CEE;">配置路径：发票云->系统管理->发票云授权配置</font>

![](https://cdn.nlark.com/yuque/0/2025/png/35533825/1759196659108-fb8a12a3-5e06-42a2-b3db-bccfdddb6ac9.png)

**配置项说明**

+ url：正式环境：<u><font style="background-color:#E8F7CF;">https://api.piaozone.com</font></u>/测试环境：<u><font style="background-color:#E8F7CF;">https://api-dev.piaozone.com/test</font></u>

<font style="color:#585A5A;">（如果使用代理跳转根据实际情况配置）</font>

+ 代理：代理的ip和端口，如“172.18.1.1:10086”，<font style="color:#DF2A3F;">没有则不配</font>
+ 是否使用原授权：新用户统一选“否”，老用户可选“是”或“否”

<font style="color:#585A5A;">（如选是，需到，系统服务云-基础资料-发票云配置中去进行企业授权）</font>

<details class="lake-collapse"><summary id="u1e8e86c2"><span class="ne-text" style="color: #585A5A">使用原有授权”为“是”与“否”的逻辑：</span></summary><ol class="ne-ol"><li id="ua99d81d6" data-lake-index-type="0"><span class="ne-text" style="color: #8A8F8D">如果选否：则通过任选其一的密钥，去判定整个购买企业买了多少个税号，如一个企业买了100个税号，正常应该生成100个企业及其对应的密钥，任选一个密钥，就会向数据库后台查询这个企业整体购买了多少个税号，当涉及服务访问的时候只要属于这100个税号里面的任意一个企业，都可以用发票云的服务</span></li><li id="u46f8c515" data-lake-index-type="0"><span class="ne-text" style="color: #8A8F8D">如果选是：一般是费报或其他模块启用了发票云，根据费报的设计，每个企业都要绑定对应的税号，即100个企业要配置100个不同的密钥，且这种控制是根据企业来的，如a企业配置了密钥，如果登陆组织为B企业，则B企业就没有发票云的使用权限（此种情况在选否的情况下就不存在），选了“是”则验证就会到费报那边去做验证，具体路径为：“系统服务云”-&gt;“基础资料”-&gt;“发票云配置”或“财务云”-“费用核算”-“发票设置”-“发票云配置”中进行配置（两者在哪里配都可以，一个东西），一般都是建议选否，但如果选了否，费报那边若启动也必须配（费报的强制逻辑），注意无论选“是”还是“否”这里必须配置上，否则会导致一些发票云公共服务不能用，如此处选了“是”费报模块是正常用的，但是如果进行邮箱推送发票，就会报错，因为邮箱推送服务，取的是这里的配置</span></li></ol><p id="u45414d68" class="ne-p"><span class="ne-text" style="color: #8A8F8D">举例：A企业在“发票云授权配置”中，配置了授权，且在“许可授权”中可以正确的显示具体的授权数量，并进行了企业授权，但在费报中点击调用发票助手，系统显示“当前组织没有发票云收票特性许可”，此类问题的产生及解决就是需要到上方说的费报中去配置每个企业的授权码，并保证你操作的组织是该授权组织</span></p><p id="ubb6b4c43" class="ne-p"><br></p></details>
+ Client id：对应企业授权信息中的<font style="color:#117CEE;">ClientID(授权账号)</font>
+ Client Secret：对应企业授权信息中的<font style="color:#117CEE;">ClientSecret(授权秘钥)</font>
+ Encrypt Key：对应企业授权信息中的<font style="color:#117CEE;">EncryKey(加密key)</font>

参考：[产品激活及参数获取](https://jdpiaozone.yuque.com/nbklz3/tadboa/sf09ttllvsbpkyae#jHRqm)收到的授权邮件中任取一个有权益企业税号的授权配置信息进行配置<font style="color:#DF2A3F;">（请注意：不要配置成租户的授权信息）</font>

填写完成后点**<u><font style="color:#117CEE;">“保存”</font></u>**后再点击“**<font style="color:#117CEE;">连接测试</font>**”如提示测试通过，则配置成功

#### 维护租户授权信息
<font style="color:#DF2A3F;">同一租户只需维护一次，后续同租户下其他企业升级数电无需重复此步骤。</font>

在数电配置页面点击【<font style="color:rgb(0,0,0);">维护租户授权信息</font>】，<font style="color:rgb(0,0,0);">打开租户授权信息维护页面。</font>

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1761126824088-df8740ee-de6b-41b2-80b0-c68a9d91a032.png?x-oss-process=image%2Fformat%2Cwebp)

<font style="color:rgb(0,0,0);">在打开页面中维护授权账号、授权密钥、加密key，分别对应授权邮件中</font>**<font style="color:#DF2A3F;">租户级别</font>**<font style="color:rgb(0,0,0);">的ClientID、ClientSecret、EncryKey</font>

**注意：**

+ <font style="background-color:rgb(255,255,255);">正式环境需维护正式订单的租户授权信息，测试环境需维护测试订单的租户授权信息。</font>

#### 维护电子税局账号
##### 添加电子税局账号

1. 在数电账号配置列表，点击蓝色企业名称，进入数电配置详情页。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1761130209827-3651fce2-aa9a-43e4-9884-2e575a57af10.png)

2. 点击**<font style="color:#117CEE;">“新增账号”</font>**按钮，在弹窗界面添加电子税局账号。

<font style="color:#8A8F8D;">（企业名称 、企业税号根据单据头自动带出，不可修改）</font>

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1753238230987-cb567acf-33b3-4de4-a677-51714981eeb1.png)

若登陆类型选择为“小号登录”<font style="color:#8A8F8D;">（仅限账号是“企享云办税小号”类的账号使用）</font>，需维护由企享云提供的“第三方app_key”和“第三方app_secret”（参考[办税小号开通及配置手册](https://jdpiaozone.yuque.com/nbklz3/tadboa/xyf6v6xcg2yx8cdk?singleDoc#)）![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1753239566827-5df39f83-570c-4f78-a4b0-8e4699a4fcde.png)

![](https://cdn.nlark.com/yuque/0/2023/png/34387723/1684810318533-446e75d0-d271-47d7-8d43-a229b640f229.png?x-oss-process=image%2Fformat%2Cwebp)

### 发票云版本低于V8.0
<font style="color:#117CEE;">路径：发票云→基础资料→企业管理→数电配置</font>

点击【升级数电发票】![](https://cdn.nlark.com/yuque/0/2023/png/12881570/1686810511599-6fc145a1-bb81-475d-9579-236de5ba3203.png?x-oss-process=image%2Fformat%2Cwebp)

选择需要升级的税号，可选的税号为<font style="color:#DF2A3F;">已完成星瀚发票云许可授权</font>、当前登录企业具备权限的企业，点击【确定】。

![](https://cdn.nlark.com/yuque/0/2022/png/12881570/1670401303532-f947e57d-00fd-46a8-ba6b-4ce8999b28b5.png)

填写收件邮箱地址<font style="color:#585A5A;">（用于接收升级数电必备信息）</font>，点击<font style="color:#117CEE;">【确定】</font>，系统将自动推送升级数电所需的信息<font style="color:#585A5A;">（激活码、操作手册）</font>至填写的邮箱。

# 四、常见问题解答（FAQ）
### <font style="color:#117CEE;">连接测试不通过如何排查？</font>
+ 确保配置的URL是可以在服务器上访问的（注不是网页访问，而是通过运维方法确保这个网址是通的）。
+ 检查是否环境与密钥配置不匹配，经常出现测试环境url配置了生产的密钥，反则反之。所有的id错误提示，如"非法的客户id"、"签名认证不通过"、"应用id不能为空"等都是密钥配置有问题导致。
+ 检查企业是否有代理服务（一般问企业运维都知道）。

### <font style="color:#117CEE;">为什么要配置这个url？</font>
发票云常见的一些公有服务（无法私有化的）都是需要通过这个url地址来进行访问，如：勾选认证、发票识别、发票查验、黑名单等，总之可以理解与税局交互的所有服务都是需要通过这个url来进行中转，该配置是发票云最基础的配置，如不配置，所有的发票云功能都不可用。

### <font style="color:#117CEE;">ERP访问税局服务的基本逻辑</font>
+ 1激活时，在我们后台存了税号跟对应的秘钥；
+ erp中绑定，对应的税号跟秘钥；
+ 当erp发起请求时，如识别服务，会带erp中的税号跟秘钥访问我们的服务器即这个URL地址；
+ 我们的服务器进行权益验证通过后，才会调用公有云服务，如识别查验、勾选、认证等。

### <font style="color:#117CEE;">使用发票云时提示“无特性分组”如何处理？</font>
+ 若发票云模块点击功能提示无特性分组，检查发票云授权配置和许可授权
+ 若费报/应付打开提示无发票云特性分组，检查费报/应付模块授权配置
