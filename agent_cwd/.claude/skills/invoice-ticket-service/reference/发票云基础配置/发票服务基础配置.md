## 公有云
+ 确定购买的发票云产品是否为“星空旗舰版”，产品购买错误则无法正常显示
+ <font style="color:rgb(0, 0, 0);">确定购买的是金蝶公有云服务，需要在订单</font>**<font style="background-color:#ffff00;">交付中</font>**<font style="color:rgb(0, 0, 0);">的状态，对应环境才会部署发票服务模块</font>
+ 订单是交付中，等待晚上部署补丁后会更新为已交付（<font style="color:rgb(0, 0, 0);">星空旗舰公有云上补丁时间为20:00</font>）
+ 订单已经交付完成，产品还是没有发票服务模块，需要提单转研发分析

# 二、基础服务云配置
## 创建行政组织
**<font style="color:#117CEE;">路径：【基础服务云】→【企业建模】→【人员部门管理】→【行政组织（部门）】</font>**

<font style="color:rgb(0, 0, 0);"> 点击行政组织（部门），进入部门列表，点击【新增】进入行政组织（部门）新增界面。  </font>

![](https://cdn.nlark.com/yuque/0/2025/png/39256605/1763086771194-3da0c8d3-2439-4b5f-b713-74f1256d431f.png)![](https://cdn.nlark.com/yuque/0/2025/png/39256605/1763086778899-4c5de833-c4d4-44db-b1b2-fae2a51204bf.png)

## 创建业务单元
**<font style="color:#117CEE;"> 路径：【基础服务云】→【企业建模】→【组织管理】→【业务单元】。</font>**

 点击“新增”，进入新增界面  

点击需配置的企业，点击企业明细，注意把职能类型设置为“核算组织”，否则发票云会选不到这个企业

## 创建人员/角色及模块授权
### 新增人员
**<font style="color:#117CEE;"> 路径：【基础服务云】→【企业建模】→【人员部门管理】→【人员】  </font>**

 点击人员后，进入人员列表，点击“新增”，调出人员新增界面  

![](https://cdn.nlark.com/yuque/0/2025/png/39256605/1763087029224-1bf66da0-6a48-44f9-8ac7-c6025fd567ac.png)![](https://cdn.nlark.com/yuque/0/2025/png/39256605/1763087035064-b4e4399b-8a30-4d7e-bb94-b714d68581e4.png)

### 授权全功能用户
**<font style="color:#117CEE;">路径： 【基础服务云】→【安全管理】→【用户权限管理】 →【全功能用户】  </font>**

进行<font style="color:#DF2A3F;">全功能授权</font>，该用户就具备了全部的系统权限，然后使用<font style="color:#DF2A3F;">全功能用户账号</font>登录星瀚后继续进行业务模块的配置

<font style="color:#2F4BDA;">（注：权限设置细分是设置是非常复杂的，这里仅解释最快捷的设置方法，具体细分详见星瀚的人员权限及组织设置）</font>

+ _<font style="color:#585A5A;">点击【添加用户】即可选择用户，添加为全功能用户。</font>_
+ _<font style="color:#585A5A;">点击【删除用户】</font>__<font style="color:#585A5A;">即可将用户取消设为全功能用户。</font>_
+ _<font style="color:#585A5A;">当全功能用户数据过多时，支持按姓名/工号进行搜索，查看目标用户是否为全功能用户。</font>_
+ _<font style="color:#585A5A;">在用户功能权限报表中查看用户权限时，若用户为全功能用户，则出现业务提示告知。</font>_

![](https://cdn.nlark.com/yuque/0/2025/png/39256605/1763087172955-ab99d674-30a5-4587-abec-3b1969377fff.png)![](https://cdn.nlark.com/yuque/0/2025/png/39256605/1763087191974-36d812e8-4e49-4265-95d2-729d2fb36458.png)

具体操作可参考：[账号及权限分配](https://vip.kingdee.com/link/s/ZTdQd) 

# 三、发票授权配置
**<font style="color:#117CEE;">路径：发票服务-系统管理-发票服务授权配置</font>**

**<font style="color:#000000;">参考页面提示，补充url、client id、Client Secret、Encrypt Key（对应授权邮件中的企业授权）</font>**

**<font style="color:#000000;">配置项填写说明：</font>**

+ <font style="color:#000000;">url：根据星瀚实际环境填入正式环境地址/测试环境地址（如果使用代理跳转根据实际情况配置）</font>
        * <font style="color:#000000;">正式环境：</font>[<font style="color:#000000;">https://api.piaozone.com</font>](https://api.piaozone.com)
        * <font style="color:#000000;">测试环境：</font>[<font style="color:#000000;">https://api-dev.piaozone.com/test</font>](https://api-dev.piaozone.com/test)
+ <font style="color:#000000;">代理：</font><font style="color:rgb(0, 0, 0);">如果有host代理，配置代理ip和端口，如：</font><font style="color:#000000;">172.18.1.1:10086（没有则置空，一般没有）</font>
+ <font style="color:#000000;">是否使用原授权：一般选择“否”</font>
+ <font style="color:#000000;">Client id（授权码）、Client Secret（授权密钥）、Encrypt Key（加密密钥）请填入任一企业授权三个值</font>
+ <font style="color:#000000;">【保存】- >点击【连接测试】，连接成功即可。</font>



<font style="color:#000000;">如</font>**<u><font style="color:#DF2A3F;">连接失败</font></u>**<font style="color:#000000;">，请根据报错原因排查具体原因，一般有两种原因：</font>

<font style="color:#000000;">A. url与授权信息不匹配，正式环境地址配正式环境授权，测试环境地址配测试环境授权。</font>

<font style="color:#000000;">B. 客户使用内网的情况下，检查是否已将发票服务域名加入网络白名单。</font>
