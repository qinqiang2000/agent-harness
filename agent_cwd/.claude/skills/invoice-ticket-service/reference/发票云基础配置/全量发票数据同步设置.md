可配置下载方案定时获取该核算组织的进销项发票归集至全票池，获取发票明细信息，用于发票的统计分析、勾选认证等、更新进项状态等操作。

+ <font style="color:rgba(0, 0, 0, 0.85);">部门管理人员：财务部门管理人员，解决了对发票数据进行统一管理和监控的痛点，能够通过设置发票数据下载方案，确保发票数据的完整性和合规性，及时掌握发票的下载情况和状态。</font>
+ <font style="color:rgba(0, 0, 0, 0.85);">公司管理层：公司高层管理岗位，解决了对公司发票数据进行整体把握和决策的痛点，能够通过发票数据下载设置，获取发票的统计分析数据，为公司的财务决策和成本控制提供依据。</font>

##### **各通道定时同步任务配置说明：**
###### <font style="color:rgb(149, 55, 52);">若为电子税局通道：</font>
① 全量发票下载：

+ 当前月：每3天下载一次当前月份开具的发票。此下载任务通常可执行成功，但仍存在异常情况导致下载资源浪费，因此对下载任务做了优化，详见以下截图：  
![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1748507224408-70c60180-6c25-4c80-bb51-a3715ad1ef5b.png)
+ 历史月：
    - 每月2号全量下载上个月销项发票；
    - 每月6号全量下载上个月进项发票；
    - RPA通道于每月月初下载上个月进销项全量发票时（2号销项全量，6号进项全量），同时作废上个月遗留的未处理及处理中的当月进销项全量任务，确保下载任务有序运行。（例如2月创建的下载2月发票的任务a仍在处理中，那么在3月下载上个月全量发票时，作废2月创建的任务a）。
    - 每月10号全量下载上个月进销项发票（给2号、6号的下载做补偿，避免下载不全），（默认不开启，需配置方可启用，进入参数配置单据列表：配置rim_download；open_replenish_last_month，1）；
    - 每月12日会检测上上月的全量数据是否下载，若没有下载，则生成任务申请下载，若下载成功，则不生成下载任务（例如3月12日会检测1月的全量数据是否下载，没有下载则申请下载；如果2月2日下载1月的销项已经成功，2月6日下载1月的进项已经成功，那么3月12日不再生成下载1月发票的任务）。
    - 其中，对于申请下载海关缴款书的：
        * 历史月中：下载上个月的海关缴款书的，下载范围由上个月扩展至上上个月+上个月填开的缴款书 ，以保证海关缴款书下载完整。
        * 当前月中：默认每3天运行一次的查询当月海关缴款书的定时任务，填开日期起始时间扩大到当月1号（即下载1号开始到T-1号开具的发票），以保证海关缴款书下载完整。

（下载数据前，无论已在票池中的发票是已验还是未验，当RPA通道下载获取到发票全量数据，满足“RPA全量能下载到的字段”且“RPA返回的字段值非空字符串”且“字段在进项全票池有”，就做数据覆盖保存--更新票池的字段，未验的发票默认更新为已验。若是票池已有字段，但RPA未获取到该字段，则不修改全票池已有的字段的值，例如全票池已经记录了查验取得的销方地址电话，但RPA下载不到销方地址电话，则不会因为RPA下载不到就清空原本已经有的值。）

② 发票状态更新：每天下当前属期的已勾选已抵扣发票，获取发票票面要素，更新发票的状态，可选择是否查验获取全票面数据。

###### <font style="color:rgb(149, 55, 52);">若为乐企通道：</font>
① 全量发票下载：

+ 当前月：一个月5次下载当前月份（从1号开始到T-1）开具的发票；
+ 历史月：
    - 每月2号全量下载上个月销项发票；
    - 每月6号全量下载上个月进项发票；
    - 每月10号全量下载上个月进销项发票（给2号、6号的下载做补偿，避免下载不全）；（默认不开启，需配置方可启用，进入参数配置单据列表：配置rim_download；open_replenish_last_month，1）；

② 增量发票下载：每天凌晨下载前四天的发票，获取全票面数据。<font style="color:#117CEE;">（注意此处的前四天是指被乐企平台归集数据的时间在前四天，而非发票的开票日期在前四天。）</font>

_<font style="color:#117CEE;">（举例说明：</font>_

_<font style="color:#117CEE;">存在发票A：开票日期：1月5日、乐企平台归集数据时间：1月6日（实际乐企的归集数据时间可能是开票当天，也可能延后几天）；</font>_

_<font style="color:#117CEE;">在1月5日执行的定时任务a中：任务查询范围1月1日-1月4日-共4天，发票A的归集数据时间为1月6日，不在1月1日-1月4日区间内，因此不会被查询到。</font>_

_<font style="color:#117CEE;">在1月7日执行的定时任务b中：任务查询范围1月3日-1月6日-共4天，发票 A 的归集数据时间为1月6日， 在1月3日-1月6日区间内，因此会被查询到，此时可以下载此发票，获取全票面数据。）</font>_

_<font style="color:#117CEE;">---区间日期的任务同1月6日，同样查询不到。</font>_

_<font style="color:#117CEE;">在1月11日执行的定时任务b中：任务查询范围1月8日-1月10日，发票 A 的异常认定时间为1月10日，在1月8日-1月10日区间内，因此会被查询到，此时可以更新此发票的异常状态。）</font>_

##### “进项状态更新的发票是否查验入库”说明：
*进项状态更新获取的发票查验入库：即所有的下载发票进入全票池，如果所下载的数电普票不在库中且设置的"进项状态更新获取的发票查验入库"，则下载成功后走查验并入库。

*进项状态更新获取的只更新状态：即所有下载的发票都不收入全票池，仅对全票池里已通过业务系统收入的发票的状态，如跟下载的发票状态不一致进行更新，例如税局下载100张发票，全票池30张发票，仅对这全票池的30张更新状态，下载的新增的70张发票不进入全票池

*进项状态更新获取的发票查验不入库：即只下载查验状态到缓存库，待业务系统实际收入发票时，先去查验缓存库去取信息。

##### 使用金税四期下载数电票版式文件的条件说明：
<font style="color:rgb(149, 55, 52);">若为电子税局通道：</font>发票服务基础资料-企业信息-收票通道需要配置“新电子发票服务平台”，全量发票数据同步下载类型为“进项-全量”，启用金税四期下载数电发票版式文件开关。即可从电子税局下载数电票版式文件，支持格式包括 PDF、OFD 和 XML。

<font style="color:rgb(149, 55, 52);">若为乐企通道：</font>发票服务基础资料-企业信息-收票通道需要配置“乐企平台”、填写了国家和地区（目前只有中国可下载）、填写了税局城市及编码，全量发票数据同步下载类型为“进项-全量”或”进项-增量“，启用金税四期下载数电发票版式文件开关。即可使用定时任务从乐企归集进项发票数据，若乐企归集（数电普票、数电专票、数电航空客票、或数电铁路客票）任务成功，且发票购方企业在发票服务基础资料有填写企业所属电子税局城市及编码的，才能生成数电票版式文件下载任务（①对于乐企归集任务失败的，不生成版式文件下载任务。②对于发票购方企业在发票服务基础资料未配置税局城市及编码的，不生成版式文件下载任务）。定时任务将调用RPA接口申请下载PDF、OFD和XML三种文件，同时星空会用OFD生成快照。

使用任一通道成功下载文件后，根据配置：[ 税局文件下载到后是否自动签收]，判断是否自动签收发票。（从税局下载源文件：若配置自动签收，则发票标为已签收，下次用户上传文件后不覆盖；若配置不签收，则发票仍为未签收状态，下次用户上传文件后覆盖。）
