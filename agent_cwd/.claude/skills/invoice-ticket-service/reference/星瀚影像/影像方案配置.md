## 影像方案配置功能设计理念概述
影像方案配置用于统一管理影像模式及其模式下的相关配置，是影像系统的重要控制中枢。

1. **<font style="color:rgb(0, 0, 0) !important;">全场景模式覆盖</font>**
    - **<font style="color:rgb(0, 0, 0) !important;">匹配模式</font>**<font style="color:rgba(0, 0, 0, 0.85) !important;">：通过结构化数据校验（如发票数量、内容一致性）降低退扫率，适合标准化程度高的流程（如集团化财务审核）。</font>
    - **<font style="color:rgb(0, 0, 0) !important;">同步模式</font>**<font style="color:rgba(0, 0, 0, 0.85) !important;">：免扫描快速生成影像，适配移动办公、人人可上传的轻量化场景（如差旅报销）。</font>
    - **<font style="color:rgb(0, 0, 0) !important;">基础模式</font>**<font style="color:rgba(0, 0, 0, 0.85) !important;">：独立运行不依赖外部系统，适合无收票接口的传统企业或临时单据采集需求。</font>
2. **<font style="color:rgb(0, 0, 0) !important;">智能化规则引擎</font>**
    - <font style="color:rgba(0, 0, 0, 0.85) !important;">参数配置支持 “场景模板” 快速调用（如 “电票自动签收模板”“纸票人工复核模板”），减少重复配置时间。</font>
    - <font style="color:rgba(0, 0, 0, 0.85) !important;">动态提示配置影响（如 “签收完成后自动提交” 开启时，需同步配置 ERP 提交状态），降低操作风险。</font>
3. **<font style="color:rgb(0, 0, 0) !important;">灵活的权限与范围控制</font>**
    - <font style="color:rgba(0, 0, 0, 0.85) !important;">支持按组织层级（全局 / 子公司）配置方案，避免跨组织规则冲突；业务单据类型过滤确保规则精准生效。</font>

### 影像方案配置主要操作说明
##### 三种影像模式说明及适用场景
###### 匹配模式	
当影像模式为“匹配”时，该方案适用“匹配签收规则”部分的参数配置。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1747358332575-263e6ead-4fc6-4591-b5f4-fab55b9f8309.png)

1. 模式说明
    - 提单人在 ERP 中创建单据并通过收票功能采集发票，收票模块保存发票结构化数据与业务单据的关联关系，并通知影像系统，同步发票数据。
    - 提单人将实物单据递交至扫描岗，由扫描岗在影像系统中采集影像。
    - 采集完成后，系统调用识别查验接口，对发票影像进行识别与查验，获取影像对应的结构化数据。
    - 系统将影像结构化数据与收票结构化数据进行匹配：匹配成功则提交至 ERP，匹配失败则停留在影像节点。
    - 审核岗在 ERP 调阅单据时，可同时调阅并查看对应的影像。
2. 模式优点
    - 对采集影像进行识别、查验并返回匹配结果，减少漏交、错交的概率。
    - 降低审核岗的人工审核工作量，退扫概率相对较低。
    - 纸质单据完整性较好，归档时既有影像文件，也有结构化数据。
3. 模式缺点
    - 对收票模块依赖度高，影像系统不能作为完全独立的系统处理业务。
    - 对后期业务开展的局限性：对于非发票类的附件暂不支持识别，这部分需要人工复核。
4. 模式适用场景
    - 有指定扫描岗。
    - 已启用收票功能或业务系统已集成收票能力，可在提单时获取发票结构化数据。
    - 需要在影像系统采集影像，并希望通过系统执行初审（二次匹配）。
5.  模式时序图 

![](https://cdn.nlark.com/yuque/0/2023/png/25465214/1701410032569-4d9ffcb0-54cc-4a77-abd2-f2628bc461a9.png)

###### 同步模式
当影像模式为“同步”时，该方案适用“同步规则”部分的参数配置。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1747358613460-e025d508-e89a-455f-b319-12f6d2bd2429.png)

1. 模式说明
    - 提单人在 ERP 中创建单据，并通过收票功能采集发票源文件。
    - 收票采集完成后通知影像系统，同步发票数据并在影像系统中生成影像。
2. 优点
    - 只需要在收票环节完成一次采集，无需在影像系统中再次扫描，整体灵活性较高。
3. 缺点
    - 同步依赖接口与网络，若同步失败，流程可能长期停留在影像“待上传”节点。
    - 对后期业务开展的局限性：同步失败时，审核岗在 ERP 调阅影像时无可用影像数据，影响审核时效性。  
4. 适用场景
    - 无固定扫描岗，允许业务人员自行上传影像。
    - 不需要在影像系统内执行扫描采集，只依赖收票端采集的电子文件。
5. 模式时序图

![](https://cdn.nlark.com/yuque/0/2023/png/25465214/1701410033058-698e7e72-5318-4fb7-b832-09f070348ebe.png)

###### 基础模式
当影像模式为“基础”时，该方案适用“基础规则”部分的参数配置。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1747358716372-7b757e15-972c-4e29-b375-0626c8afc66c.png)

1. 模式说明
    - 提单人在业务系统中完成提单后，直接在影像系统中通过扫描页面采集影像（只采集单据影像，不对影像进行识别处理。）
    - 采集并上传成功后，影像系统通知业务系统：相关单据的影像已上传。
2. 优点
    - 影像系统可以作为相对独立的系统采集影像，与外部系统对接较少、依赖较低，灵活性较好。
3. 缺点
    - 无法判断单据有效性，可能产生冗余数据。
    - 仅采集影像不做识别处理，系统中只有影像，没有对应的发票结构化数据。
    - 对后期业务开展的局限性：增加审核岗复核影像的工作量。且影像不合规导致退扫的概率相对较高，不利于后续文件归档信息的完整性。
4. 适用场景
    - 客户有指定扫描岗，没有用收票系统。需要在影像系统中统一采集“发票 + 附件”影像的场景。
5. 模式时序图

![](https://cdn.nlark.com/yuque/0/2023/png/25465214/1701410031892-0ea13e63-10a6-4800-81e2-746410473134.png)



##### 配置影像模式
1. ******<font style="color:rgb(33, 33, 33);">匹配签收规则</font>**

![](https://cdn.nlark.com/yuque/0/2026/png/38481329/1767923883507-23ec4dfc-b950-4d63-a82b-e1dc77e1c7c4.png)

+ **<font style="color:rgb(33, 33, 33);">签收方式：</font>**<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。匹配模式下，当发票的影像结构化数据与收票结构化数据一致时签收成功。配置【系统签收】则数据一致时自动签收；配置【人工签收】则需人工审核数据一致操作签收。</font>
+ **<font style="color:rgb(33, 33, 33);">发票签收配置</font>**<font style="color:rgb(33, 33, 33);">：匹配模式下哪种类型的发票需要进行匹配；可选“纸票”“纸票+电票”或为空。</font>
+ **<font style="color:rgb(33, 33, 33);">附件签收配置：</font>**<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。匹配模式下是否对附件进行匹配签收，可配置“不支持”“纸质附件”“电子附件”“纸质附件+电子附件”。</font>
+ **<font style="color:rgb(33, 33, 33);">单封面单据提交：</font>**<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。匹配模式下单据采集是否允许单封面提交，设置为校验则单封面无发票和附件的单据不可提交；反之设置为不校验则可提交。</font>
+ **<font style="color:rgb(33, 33, 33);">已提交单据请允补充影像：</font>**<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。匹配模式下已提交单据是否允许进行附件补扫，配置【支持】则已提交单据在采集界面可进行封面和采集附件提交，提交后影像数量是叠加的；配置【不支持】则已提交单据在采集界面采集封面会提示单据已提示不可重复采集。</font>

![](https://cdn.nlark.com/yuque/0/2026/png/38481329/1767924728138-b5531c77-de82-4aae-9d54-c9aef30b73fd.png)

+ **<font style="color:rgb(33, 33, 33);">签收完成后自动提交：</font>**<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。匹配模式下影像与收票数据一致签收成功后单据是否自动提交。配置【自动提交】则签收成功后系统自动提交影像单据向后流转；配置【手工提交】则签收成功后需人工提交影像单据。</font>
+ **<font style="color:rgb(33, 33, 33);">签收提交ERP的状态：</font>**<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。配置影像单据哪种状态允许向后流转提交业务系统。</font>
+ **<font style="color:rgb(33, 33, 33);">匹配纯电单据执行同步任务状态限制：</font>**<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。系统默认所有状态全选，可配置影像单据状态在哪种情况下执行同步业务系统状态信息。</font>
+ **<font style="color:rgb(33, 33, 33);">匹配数量管控：</font>**<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。匹配模式下是否对发票数量进行签收。配置【支持】则只有数量和内容都一致才能签收成功；配置【不支持】则影像采集的文件包含提单人文件就能签收成功。</font>
+ **<font style="color:rgb(33, 33, 33);">按票签收：</font>**<font style="color:rgb(33, 33, 33);">匹配模式下，配置按发票文件维度签收。配置【支持】则不需要指定操作影像单据编号直接采集发票进行签收；配置【不支持】则必须指定影像单据编号才能采集文件</font>
2. **<font style="color:rgb(33, 33, 33);">同步规则</font>**

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1747374811292-62f62743-8087-43ac-8248-ffe25ddfcd8a.png)

+ **同步完成后允许补扫影像**：<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。同步模式下收票数据同步到影像后是否允许进行补扫影像；若配置【允许】则影像单据停留影像系统需人工操作提交向后流转。</font>
+ **同步补扫影像进行结构化数据提取**：<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。同步模式允许同步完成后补扫影像时是否对补扫的影像提取结构化数据。</font>
+ **可执行同步任务的影像单据状态**：<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。系统默认所有状态全选，可配置影像单据状态为哪种情况下执行同步业务系统状态信息。</font>
+ **同步收票文件范围**：<font style="color:rgb(33, 33, 33);">此项为必填项可以通过下拉框选择。同步模式下单据上的哪些文件类型同步生成影像。</font>
