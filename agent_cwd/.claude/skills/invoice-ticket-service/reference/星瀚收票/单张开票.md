## 单张开票功能设计理念概述
### 功能概述
发票服务除了提供基于单据的拆分合并的复杂业务场景的开票功能外，还提供方便快捷的独立开票的能力，适用于一些不强关联业务单据的开票场景。支持用户自定义发票种类、征税模式、开票抬头和明细信息。

### <font style="color:rgba(0, 0, 0, 0.85);">目标客户以及客户解决核心痛点问题</font>
1. <font style="color:rgba(0, 0, 0, 0.85);">操作使用人员：财务或相关开票岗位，解决了在一些不强关联业务单据的开票场景中，快速、准确地进行单张发票开具的痛点，能够自定义发票种类、征税模式、开票抬头和明细信息，提高了开票的灵活性和效率。</font>
2. <font style="color:rgba(0, 0, 0, 0.85);">部门管理人员：财务部门管理人员，解决了对无单据开票进行管理和监控的痛点，能够确保开票操作的合规性和准确性，同时可以通过购方抬头的自动保存功能，方便后续开票操作。</font>
3. <font style="color:rgba(0, 0, 0, 0.85);">公司管理层：公司高层管理岗位，解决了对无单据开票管理进行监督和决策的痛点，能够及时了解开票情况，为公司的财务管理提供支持，确保公司的税务合规。</font>

### 单张开票主要操作说明
#### 单张直接开票
**步骤1：**完善开票基本信息。填写包括此次要操作的组织、“发票类型”和“征税方式”等，由创建者根据实际情况和需要进行设定。选择票种，不同特殊票种有不同的填写项目，请根据实际要求填写发票信息。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1750818271675-b984d617-4a26-4356-9011-4709eb8a8cc9.png)

差额征税的备注处理功能：将征税方式从普通征税切换为差额征税时，不清空原备注，将扣除额拼接在原备注前方。

（1）将征税方式从普通征税切换为差额征税（含全额、差额）时，不清空原单据备注，填写扣除额之后，将扣除额拼接在发票备注前方，扣除额：XX元。原备注。 示例：原备注为A，普通征税切换为差额征税后填写扣除额10元，最终备注为 扣除额：10元。A （2）将征税方式从差额征税（含全额、差额）切换为普通征税时，删除字段头的【扣除额：XX元。】文字。

**步骤2：**完善购方抬头信息。在名称的下拉菜单中选择或输入（支持名称的模糊搜索），税号、地址等信息可以自动匹配出。在基础资料基础信息设置中开启购方抬头的自动保存功能，可以将单张开票使用过的购方抬头信息自动保存起来，方便后续快速选择开票。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1750818357968-4a8b9d3d-c969-4139-bd10-8c6d0311e65c.png)

**步骤3：**完善项目信息。点击“增加明细”按钮，添加新的项目。在项目名称的框内按ENTER键即可在菜单中选择已有商品，填写规格型号、单位、单据等信息。确定税率后，税额可由系统计算出。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1750818383381-996c24dc-8563-47ba-827d-d7dcdbf4a8a3.png)

**步骤4：**填写备注、收款人、复核人的其他销售方信息。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1750819308563-e3758915-6da2-4167-90fc-4b8b292527a5.png)

开具数电发票，当选择征税方式为差额征税-差额开票时，备注会展示录入的扣除额内容，例如：扣除额：5元。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1744192332714-335ebe29-fb38-4c59-9a3b-8813928b1b3b.png)

**步骤5：**点击“更多”按钮，还可填写购方经办人信息。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1750818430808-c001affa-8bd5-42c8-bffd-405184428ee6.png)

**步骤6：**点击“开票”按钮。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1750818452539-24a3bce2-8781-48e9-9783-57c285a23f4c.png)

#### 保存开票数据至开票过程监控
“保存”按钮，按照第一点【单张直接开票】步骤1-步骤5填写开票数据后，点击“保存按钮”可将发票数据保存至开票过程监控，保存成功提示保存成功，并清空当前页面发票数据。

注意：符合开票条件后，才允许点击“保存”按钮。保存至开票过程监控列表后，数据来源为单张开票。

![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1750819277245-230c07d9-5c0b-4af6-b567-6fa2d2ae3648.png?x-oss-process=image%2Fformat%2Cwebp)

保存至开票过程监控后，可通过下述配置控制数据是否需要审批，未配置时为无需审批。

开发平台搜索【参数配置单据】，选中列表后点击预览，设置配置项（若无配置项，需要点击新增）：<font style="color:#000000;">待开发票默认审核状态配置（开票申请单下推的待开该配置不生效）</font>

+ <font style="color:#000000;">配置项类型</font> : <font style="color:#000000;">sim_not_approval_required</font>
+ <font style="color:#000000;">配置项key</font> : <font style="color:#000000;">sim_invoice_wait</font>
+ <font style="color:#000000;">配置项值</font> : <font style="color:#000000;">A暂存，D无需审批（例：D）</font>

<font style="color:#000000;">更多配置项见：</font>[https://www.yuque.com/piaozone/implement/egoz20](https://www.yuque.com/piaozone/implement/egoz20)

#### 发票预览：开票前预览发票
“发票预览”按钮，按照第一点【单张直接开票】步骤1-步骤5填写开票数据后，点击“发票预览”调用版式文件生成服务生成样式，按照已填写内容生成发票，票面展示样票字样，超过一屏内容时，支持滚动展示。**该预览图仅供开票前预览非税局真实文件**。

发票预览时，需要满足开票条件后才允许预览，例如填写必填字段（例如金额、税率）后再进行预览，未填写时不可预览并出现报错提醒。
