## 配置路径

在合规性校验配置列表点击“新增”按钮即可开始配置

![](https://cdn.nlark.com/yuque/0/2026/png/38481329/1767579713921-ad01dd45-1b2c-49f8-8665-698b18a26201.png)

## 配置项说明
### 基础信息
编辑新增的合规性的方案名称，并选择勾选应用此方案的业务单据类型以便对特定业务单据进行合规性校验。在业务<font style="color:rgb(26, 26, 26);">单据类型中选择此条合规性配置的适用单据，如不选任何单据则为所有单据都生效。若单据类型列表中没有符合业务的单据类型可在【收票资料维护】-->【单据类型管理】新增对应的业务单据类型。</font>

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748593470088-22844599-478d-419b-82de-223cccfc97db.png)

### 基础管控
基础管控配置中可选的提示类型有严格管控、中度警示、轻度提醒、不控制四种。

<font style="color:#DF2A3F;">1、严格管控</font>

配置严格管控后，在发票助手或者发票签收上传不符合可导入条件的发票会有：

错误提示（禁止导入）：XXX（原因） ，在发票助手被严格管控拦截的发票不允许导入单据，在签收管理被严格管控拦截的发票左侧不合规张数+1。

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1723174636439-2b9d8756-96db-434e-bad0-d874b535cd71.png)

<font style="color:#DF2A3F;">2、中度警示</font>

配置红框警示后，在发票助手或者发票签收上传不符合可导入条件的发票会有：

中度警示：XXX（原因）

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1723174810274-7f54d80d-b720-4a4a-83af-d55b7489e1b6.png)

<font style="color:#DF2A3F;">3、轻度提醒</font>

配置黄框警示后，在发票助手或者发票签收上传不符合可导入条件的发票会有：

轻度警示：XXX（原因）

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1723174866318-55c02e56-c208-4f06-a36e-09041db63450.png)

<font style="color:#DF2A3F;">4、不控制：</font>配置不控制后，在发票助手或者发票签收上传不符合可导入条件的发票不会有提示。

**基础管控配置项说明：**

+ **发票抬头与当前企业名称是否一致**：发票购方抬头与单据配置的抬头不一致，此项为必填项可以通过下拉框选择。
+ **忽略特殊符号差异：**针对抬头校验上增加的是否忽略特殊符号的配置，可选值为：空格（企业名称首尾空格）、空格（所有空格）、中英文括号、中英文破折号。支持为空，支持多选。默认全选。例如选择空格（所有空格），进行抬头校验时会忽略空格。
+ **发票税号与当前企业税号是否一致**：发票购方税号与单据配置的抬头不一致，此项为必填项可以通过下拉框选择。
+ **非增值税发票手工修改**：是否支持导入时点编辑手工修改过信息的发票，原因是因为非标准发票很多通过收票助手收入的时候OCR识别不准，一般可以允许对识别错误的信息进行修改，此处建议不控制，否则会导致大量票导入不进来，此项为必填项可以通过下拉框选择。  
发票服务V7.0.18增加配置：在合规性校验-<font style="color:rgb(0,0,0);">个性化配置中，增加“非增值税发票手工修改”，支持按发票类型来校验是否手工修改，并按控制级别给出对应提示，其余未配置发票类型不控制。个性化配置配了手工修改条件后，基础管控和个性化配置的校验项同时校验。</font><font style="color:#DF2A3F;">如需按发票类型控制非增值税发票手工修改，需将基础管控中的“非增值税发票手工修改”改为不控制。</font>
+ **发票真伪**：通过识别查验判断发票真伪，目前可查验的票种包括：数电普票、数电专票、数电发票（航空运输电子客票行程单）、数电发票（铁路电子客票）、数电发票（机动车销售统一发票）、数电发票（二手车销售统一发票）、电子普通发票、电子专用发票、纸质普通发票、纸质专用发票、普通纸质卷票、机动车销售发票、二手车销售发票、通行费电子发票。
+ **发票类型**：选择需要进行合规性管控的发票类型，此项为必填项可通过下拉框选择。
+ **重复报销**：校验同一张发票在系统内是否已被报销单全额使用。重复报销会提示XX发票被XX单据引用的提示语，此项为必填项可通过下拉框选择。
+ **电子发票是否上传源文件**：在非全电的业务下，税局要求财务需保存电子发票的源文件，即系统在导入PDF 或OFD的时候会检查文件是否被篡改，若未篡改，则认为是源文件，此项为必填项可通过下拉框选择。

       星瀚地址formId=后加上rim_config跳转可进入配置页面，电子发票是否上传源文件校验项可选择“基础校验”和“严格验签”。校验失败的提示为：

       严格验签：所上传发票非源文件，文件中无有效电子签章，请联系开票方确认。

       基础验签：所上传发票非源文件，文件非原生格式，请联系开票方确认。  
![](https://cdn.nlark.com/yuque/0/2025/png/35533825/1749957923932-57b16c30-7f97-4ec2-8a8b-9db769497ede.png)

+ **发票作废**：作废的发票是否允许报销，此项为必填项可通过下拉框选择。
+ **发票红冲**：原开的发票有误或因为其它原因需更正，需要重新开具的发票调整帐目，此项为必填项可通过下拉框选择。
+ **个人发票**：是否允许上传个人抬头的发票，支持校验增值税发票（含数电机票、数电火车票、数电机动车销售统一发票），此项为必填项可通过下拉框选择。
+ **发票串号**：针对纸质普票、纸质专票校验打印机机打号码与发票上印的号码是否不一致。发票串号一般都是开票发打印错误，需退回重开，串号是指指打印机机打号码与纸质发票上印的号码不一致，一种是真的错了，那么就要重开，另外一种是识别错了，可以通过编辑来手工修改，此项为必填项可通过下拉框选择。

![](https://cdn.nlark.com/yuque/0/2023/png/26098688/1685771985282-90e7cafe-a443-456f-895e-815537f18b75.png)

+ **发票是否有印章**：对票面上的单位盖章区域进行检测，检测不到印章则进行控制，纸质发票无发票章，此项为必填项可通过下拉框选择。
+ **敏感词校验**：在【发票服务】-【基础资料】-【票据预警数据设置】-【敏感词列表】新增敏感词，上传增值税发票的商品名称含有敏感词即提示，此项为必填项可通过下拉框选择。
+ **自定义黑名单校验：**在【发票服务】-【基础资料】-【票据预警数据设置】-【黑名单列表】新增销方黑名单，上传增值税发票销方名称符合条件即提示，此项为必填项可通过下拉框选择。
+ **税务黑名单校验**：国家公示的重大税收违法失信纳税人，取的是公共服务的数据无需设置清单，上传增值税发票销方名称符合条件即提示，此项为必填项可通过下拉框选择。
+ **专票地址电话与当前企业是否一致:**上传增值税发票与【发票服务】-【基础资料】-【企业管理】中的配置的默认企业抬头信息比较，不一致即提示。建议不要设置，这里输入的信息并不严谨，如设置控制可能导致很多发票无法导入，此项为必填项可通过下拉框选择。
+ **专票开户行账号与当前企业是否一致:**上传增值税发票与【发票服务】-【基础资料】-【企业管理】中的配置的默认企业抬头信息比较，不一致即提示，此项为必填项可通过下拉框选择。
+ **销方企业抬头一致校验：**校验销方企业抬头，比对单据和发票上的销方企业抬头是否一致，此项为必填项可通过下拉框选择。
+ **销方企业税号一致校验**：校验销方企业税号，比对单据和发票上的销方企业税号是否一致，此项为必填项可通过下拉框选择。
+ **数电普票专票票面关键信息比对**：对比发票文件识别或解析的关键信息（总不含税金额、税额、价税合计、购销方名称税号、开票日期、发票号码）和税局查验返回数据是否一致，防止发票被员工篡改。

### 报销期限校验
可设置报销期限（开票时间多少天后的可以报销），是否允许跨年开关和第二年截止报销期限。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748593511409-4398098a-2cf5-4cdd-a169-cc0a926ef32f.png)

报销期限、是否允许跨年、第二年报销截至月份，三个参数取交集进行校验。

示例：

当前时间：2021-01-31

参数设置：报销期限：365； 是否允许跨年：是；第二年报销截至月份：2月；

校验逻辑：2021-2-28之前可以报销开票时间为2020-3-3号以后的发票。 2021-2-28以后就不能再报销开票时间为2020年的发票。

### 发票连号校验
![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748593533910-fb20ab8a-623a-49b5-a761-67ca3ab77634.png)对发票是否连号进行校验，注意连号的校验是判定是否绑定单据号，如 1 2 3 号发票，2 3号已经在全票池了，但没有被单据引用，则导入1时无提示，如2 3 已被引用，则提示连号（见下图），<font style="color:#FF0000;">注意连号设置中的“发票类型”要选中对应票种，为空不生效。</font>

<font style="color:rgb(0, 0, 0);">填写举例：连号范围填写10 ，就是发票号码末尾相差十以内的票都会提醒。采集发票号码050，那么连号范围校验的是040-060区间。</font>

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1723174970067-cc1ab18b-32a5-4721-bae7-455bcd1c3d5e.png)

### 文件格式校验
![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748593563703-40825eba-565a-48b1-a762-bbce74697ce3.png)

文件格式校验是检验上传的发票文件的格式是否符合要求，主要操作如下图：

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748593591430-a6b26ff1-0433-4c50-9556-f3967792ffe3.png)

点击“增行”后，在文件格式校验界面选择要校验的发票类型，选择好要控制的级别，以及选择校验的文件类型对上传的发票文件进行校验，见下图。例如：对数电普票设置的文件类型为图片，控制级别为严格管控，则不允许上传图片格式的数电普票文件。

+ 按采集时间先进先抵：<font style="color:rgba(0, 0, 0, 0.85);">按照进项发票或其他抵扣凭证被采集到系统中的时间顺序，先采集的先进行抵扣</font>
+ 按开票时间先进先抵：<font style="color:rgba(0, 0, 0, 0.85);">依据发票的开票日期，先开具的发票对应的进项税额先进行抵扣</font>

<font style="color:rgba(0, 0, 0, 0.85);">此两种取数方式需根据自己的需要进行选择。</font>

<font style="color:#ED740C;">V7.0.18 版本支持配置自动撤销入账，产品出厂默认不配置“撤销入账”方案，需用户手工新增。</font>

配置后，将启用定时任务【定时任务见本文3.3.4节手工执行自动入账任务说明】执行自动撤销税局入账，默认每天一次，客户可自行修改调度计划频次。定时任务将自动筛选全票池的发票，校验发票数据符合税局通道撤销税局入账状态的字段要求，提交到税局通道（乐企或新电子发票服务平台RPA）撤销入账；并定时查询撤销入账的任务结果，更新发票的税局入账状态。

<font style="color:rgb(0, 0, 0);">全票池将记录自动入账的重试次数，默认6次，超过不再重试，只能手工操作入账或撤销入账。</font>

<font style="color:#000000;">撤销入账只能对全票池中，税局入账状态为“已入账(企业所得税税前扣除)”“已入账(企业所得税不扣除)”撤销入账，其他入账状态及时符合业务筛选条件，该发票撤销入账任务也是失败：发票未入账或有其他入账任务处理中。</font>

**<font style="color:rgb(149, 55, 52);">若为电子税局通道：</font>**

① 全量发票下载：

+ 当前月：每3天下载一次当前月份开具的发票。此下载任务通常可执行成功，但仍存在异常情况导致下载资源浪费，因此对下载任务做了优化，详见以下截图：  
![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1748507224408-70c60180-6c25-4c80-bb51-a3715ad1ef5b.png)
+ 历史月：
    - 每月2号全量下载上个月销项发票；
    - 每月6号全量下载上个月进项发票；
    - RPA通道于每月月初下载上个月进销项全量发票时（2号销项全量，6号进项全量），同时作废上个月遗留的未处理及处理中的当月进销项全量任务，确保下载任务有序运行。（例如2月创建的下载2月发票的任务a仍在处理中，那么在3月下载上个月全量发票时，作废2月创建的任务a）。
    - 每月10号全量下载上个月进销项发票（给2号、6号的下载做补偿，避免下载不全），（默认不开启，需配置方可启用，进入参数配置单据列表：配置rim_download；open_replenish_last_month，1）；
    - 每月12日会检测上上月的全量数据是否下载，若没有下载，则生成任务申请下载，若下载成功，则不生成下载任务（例如3月12日会检测1月的全量数据是否下载，没有下载则申请下载；如果2月2日下载1月的销项已经成功，2月6日下载1月的进项已经成功，那么3月12日不再生成下载1月发票的任务）。
    - 其中，对于申请下载海关缴款书的：
        * 历史月中：下载上个月的海关缴款书的，下载范围由上个月扩展至上上个月+上个月填开的缴款书 ，以保证海关缴款书下载完整。
        * 当前月中：默认每3天运行一次的查询当月海关缴款书的定时任务，填开日期起始时间扩大到当月1号（即下载1号开始到T-1号开具的发票），以保证海关缴款书下载完整。

（下载数据前，无论已在票池中的发票是已验还是未验，当RPA通道下载获取到发票全量数据，满足“RPA全量能下载到的字段”且“RPA返回的字段值非空字符串”且“字段在进项全票池有”，就做数据覆盖保存--更新票池的字段，未验的发票默认更新为已验。若是票池已有字段，但RPA未获取到该字段，则不修改全票池已有的字段的值，例如全票池已经记录了查验取得的销方地址电话，但RPA下载不到销方地址电话，则不会因为RPA下载不到就清空原本已经有的值。）

② 发票状态更新：每天下当前属期的已勾选已抵扣发票，获取发票票面要素，更新发票的状态，可选择是否查验获取全票面数据。

_<font style="color:#117CEE;">后续预计更新为每天上午10点下载前四天的发票，基本可以做到前一天的发票，在今天可以下载到，具体的示例，将在功能发布后补充说明。</font>_

_<font style="color:#117CEE;">（举例说明：</font>_

用于配置从税局下载发票的频率。

**“进项状态更新的发票是否查验入库”说明：**

+ 进项状态更新获取的发票查验入库：即所有的下载发票进入全票池，V7.0.13版本支持如果所下载的数电普票不在库中且设置的"进项状态更新获取的发票查验入库"，则下载成功后走查验并入库。
+ 进项状态更新获取的只更新状态：即所有下载的发票都不收入全票池，仅对全票池里已通过业务系统收入的发票的状态，如跟下载的发票状态不一致进行更新，例如税局下载100张发票，全票池30张发票，仅对这全票池的30张更新状态，下载的新增的70张发票不进入全票池
+ 进项状态更新获取的发票查验不入库：即只下载查验状态到缓存库，待业务系统实际收入发票时，先去查验缓存库去取信息。

下载开始时间默认为第一次启用全量下载方案往前一年的时间，下载结束时间为最近一次进行下载的时间，选择金税四期是否下载数电票版式文件。

**使用金税四期下载数电票版式文件的条件说明：**

<font style="color:rgb(149, 55, 52);">若为电子税局通道：</font>发票服务基础资料-企业信息-收票通道需要配置“新电子发票服务平台”，全量发票数据同步下载类型为“进项-全量”，启用金税四期下载数电发票版式文件开关。即可从电子税局下载数电票版式文件，支持格式包括 PDF、OFD 和 XML。

<font style="color:rgb(149, 55, 52);">若为乐企通道：</font>发票服务基础资料-企业信息-收票通道需要配置“乐企平台”、填写了国家和地区（目前只有中国可下载）、填写了税局城市及编码，全量发票数据同步下载类型为“进项-全量”或”进项-增量“，启用金税四期下载数电发票版式文件开关。即可使用定时任务从乐企归集进项发票数据，若乐企归集（数电普票、数电专票、数电航空客票、或数电铁路客票）任务成功，且发票购方企业在发票服务基础资料有填写企业所属电子税局城市及编码的，才能生成数电票版式文件下载任务（①对于乐企归集任务失败的，不生成版式文件下载任务。②对于发票购方企业在发票服务基础资料未配置税局城市及编码的，不生成版式文件下载任务）。定时任务将调用RPA接口申请下载PDF、OFD和XML三种文件，同时星瀚会用OFD生成快照。

使用任一通道成功下载文件后，根据配置：[ 税局文件下载到后是否自动签收]，判断是否自动签收发票。（从税局下载源文件：若配置自动签收，则发票标为已签收，下次用户上传文件后不覆盖；若配置不签收，则发票仍为未签收状态，下次用户上传文件后覆盖。）

# <font style="color:rgb(38, 38, 38);">九、公有云数据同步</font>
## 功能概述
通过定时任务，将发票服务公有云的发票数据同步至星瀚发票服务。

原使用发票服务AWS公有云产品的用户，可通过定时任务或手工同步将发票数据（发票票面信息、发票报销状态、发票认证状态等）同步至星瀚发票服务。

AWS切换星瀚的用户，只需配置此处，无其他数据迁移相关配置。

## 操作步骤
**<font style="color:#117CEE;">路径：【发票服务】→【基础资料】→【公有云数据迁移】</font>**

### 进项同步/销项同步
**步骤1：**点击“进项同步”或“销项同步”并选中要同步的数据。![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748915725129-219caba8-202c-470e-a4d4-df1193dd9337.png)![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748915764947-c1de98fd-0558-4fa3-85d5-9b71fa237e5d.png)  
**步骤2：**设置数据时间，<font style="color:#000000;">如"2024-07-22",则同步"2024-07-22“新增或修改的数据, 为空全量同步，全量同步数据量比较大耗时较长，请谨慎使用。</font>点击同步按钮即可在公有云数据迁移列表中看到该数据的同步结果和同步信息。![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748915941672-0bba6365-e05b-40db-ad49-796701d204af.png)

还可以选择按单据同步数据，选中要同步数据的组织并点击”按单据同步“进入到报销单同步页面，输入该组织下的单据编号也可以同步。![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748915981255-17b995c9-e4bc-402d-9b94-61c0846c9ca4.png)

支持配置从AWS中不同步到星瀚进项全票池的字段，在开发平台-发票服务-系统管理-云应用参数配置-参数配置单据  增加配置项

 配置项key  ： rim_config_nonsync_fields  

 配置项值  ：发票主表中不从AWS同步的字段名，支持字段有：“entryAmount”,“ outputAmount”,“deductionFlag”，“outputReason”,不在范围内的字段不生效，配置多个字段时使用逗号分隔

### 进项定时同步设置
1. <font style="color:#000000;">初次使用请全量同步：系统上线做一次全量同步，选择对应的组织，并把“数据时间”清空，点击“同步”退出返回列表查看同步结果（此处企业不用新增，只要在费报处配置过授权id的企业都会显示）。</font>
2. <font style="color:#000000;">设置定时同步：在 【系统服务云】-->【调度中心】-->【调度计划】-->【发票服务】-->【收票管理】，找到“rim_InvoiceHisDataTask_SKDP_S”（ 定时下载发票历史数据）-每天增量同步当天数据，注意：“rim_InvoiceHisDataFailJob_SKDP_S”（</font> <font style="color:#000000;">历史发票同步失败重试）-增量同步失败，第二天会再次同步失败的组织，自动重试3次，3次仍然失败，请手工同步或检查网络或配置是否正确。</font>

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748916011055-687601f3-ec7c-405f-b997-0b1111a49101.png)

3. <font style="color:#000000;">公有数据与下载数据不一致情况：收票管理配置-预览，检查以下配置：“全票池显示未查验发票”为是，“全票池不展示未用发票”为是，“同步未用发票”为是，星瀚收票管理票池数据才会和AWS商家平台台账数量保持一致。</font>

![](https://cdn.nlark.com/yuque/0/2023/png/26098688/1683872957054-2420d809-b39f-4a32-8594-3bac1acac029.png)

4. <font style="color:#000000;">同步不成功排查思路：</font>
    1. <font style="color:#000000;">在费用核算-发票服务配置 配置号组织对应的clientid正确，且切换为正确的组织。</font>
    2. <font style="color:#000000;">先看下是否全部发票同步，日期为空是同步所有发票，有日期的是同步期间内AWS采集的发票。</font>
    3. <font style="color:#000000;">开发平台搜索：收票管理配置-预览，检查“同步公有云数据”栏目（一般不需要检查，默认可同步）。</font>
5. **<font style="color:#DF2A3F;">V7.0.15 版本更新：</font>**增加全局配置【同步AWS票夹数据至个人票夹】默认为关闭；开启后，当进行AWS数据同步时，一起将小程序的票夹数据同步至星瀚对应用户的个人票夹中（需在星瀚用户信息维护与微信授权一致的手机号）

### 单据是否可抵扣写入抵扣用途
**说明：**开启后单据传发票服务的是否抵扣写入票池抵扣用途字段，关闭后写入票池是否可抵扣字段

**默认值：**否

### 是否允许单据修改是否可抵扣
**说明：**部分允许指业务将可抵扣的票改为不可抵扣，但不允许将不可抵扣的票改为可抵扣

**默认值：**完全允许

### 税局文件下载到后是否自动签收
**说明：**系统判断进项发票的税局版式文件是否已经入库，如果已经入库则自动将签收状态更新为“已签收”

**默认值：**是

### 电票是否自动签收
**说明：**针对基础资料中发票类型是否电票=是的发票，上传后默认已签收

**默认值：**否

### 全票池是否展示税局下载的发票
**说明：**数据来源为“税局下载”的发票是否需要在全票池展示

**默认值：**是

### 发票是否归属个人票夹
**说明：**通过已勾选通道上传发票后，是否自动将发票归属到操作用户的个人票夹

**默认值：**是

**功能归属设置：**发票签收/识别查验外部接口/发票查验

### 发票文件导出名称设置
**说明：**在进项全票池导出发票文件时，下载发票文件的命名格式

**默认值：**发票代码_发票号码
