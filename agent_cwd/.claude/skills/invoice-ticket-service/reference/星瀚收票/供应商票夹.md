## 供应商票夹功能设计理念概述
### 功能定位
**供应商票夹**用于管理各个供应商开具的发票，这些发票由各个供应商开具并通过邮件发送给企业。  
典型使用场景：企业可以配置一个或多个公共收票邮箱，用于集中接收供应商发送的发票邮件，系统再通过“供应商票夹”功能**定时或手工从公共邮箱取票、解析发票，并按供应商维度归集展示**。

主要面向角色：

+ AP 会计 / 费用会计（日常收票、对账）
+ 税务专员 / 发票管理员（进项发票合规管理）

### 设计理念与使用价值
+ **统一入口管理来自供应商发票**：将“供应商发票 → 企业进项发票”打通，企业只需维护一个或少数几个公共收票邮箱，即可集中接收所有供应商开具的发票。
+ **按供应商维度归集展示：**左侧供应商列表基于发票中的销方信息自动归集，右侧展示该供应商对应的企业进项发票，方便对账和供应商结算管理。
+ **自动从邮箱取票，降低人工收票工作量：**通过“供应商邮箱配置+定时任务”形成完整的自动取票链路，减少手工下载、上传发票的环节。
+ **可配置的过滤与回执机制：**支持对邮件提取条件进行配置，避免无关邮件进入发票处理流程；支持在发票处理完成后自动给供应商回执发票收取结果。
+ **全流程可追踪的任务监控：**从取票任务批次，到单封邮件处理，再到每一个附件/发票的解析结果，均可分层查看处理状态和失败原因，便于运维与问题追溯。

### 供应商票夹主要操作说明
#### 按供应商查看企业进项发票
**操作目标**：  
以“供应商（发票销方）”维度查看企业已接收到并入库的进项发票，掌握其验真、签收、勾选等处理状态。

**操作步骤**：

1. 在左侧供应商列表：
+ 可直接点击“全部”查看所有供应商发票；
+ 或在“搜索供应商名称”中输入关键字，点击列表中的具体供应商名称。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763607716486-f18728ea-52ec-486b-8834-567430c32369.png)

2. 右侧发票列表将展示该供应商对应的企业进项发票，包括**采集时间、查验状态、勾选状态、价税合计、销方名称/税号、发票号码**等字段。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763607696846-e4b64835-7d5d-4a7a-a4e3-b6fb3d245dcc.png)

#### 新增供应商邮箱配置
**操作目标**：

1. 为企业新增一个收票邮箱配置，用于统一接收供应商发送的发票邮件，并作为后续自动取票的基础。
2. 在邮箱取票成功后，给发件人回执（告知供应商已收到发票）。

**操作步骤**：

1. 在【供应商票夹】页面点击【邮箱配置】，进入“供应商邮箱配置列表”。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763607743706-3c022372-ab0d-4125-a09a-fbc6c7940dc3.png)

2. 点击【新增】，打开“供应商邮箱配置”页面。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763607762090-f1e5a316-b6d6-4e41-a57b-4cc852707771.png)

3. 在“基本信息”分组中：
+ 确认或选择创建组织；
+ 输入邮箱名称，如“总部采购收票邮箱”；如开启邮件自动回复，将使用邮箱名称作为发件人名称。
+ 填写邮箱地址，即对外公布给供应商的公共收票邮箱。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763607994180-0f488f50-f231-4f60-8195-875d35cf033c.png)

4. 在“收件配置”分组中根据邮箱服务提供方提供的信息，依次填写：
+ IMAP服务器；
+ 授权码（或密码）；
+ 端口号；
+ 根据要求选择是否启用SSL/TLS加密。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763608047923-79991d9d-c8d8-43d8-92ce-3934c98d79e3.png)

5. 如希望只处理特定规则的发票邮件，在“邮件提取规则”中点击【添加条件】，过滤条件后，提取邮件内文件时将按规则提取，不满足条件的文件不提取。可选字段如下：
    - 邮件主题：文本，需手工输入值；
    - 附件文件名称：文本，需手工输入值；
    - 文件类型：可选zip、PDF、OFD、XML、图片、excel；支持多选。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763608099548-8a003867-1b3b-4371-b587-243eaad90152.png)

6. 若希望系统在每批发票处理完成后自动给发件人回执（告知供应商），在“邮件回复配置”中开启【**是否自动回复发票处理结果】**。

开启回复时，将显示“SMTP发件配置”和“发件模板配置”，配置见下一小节。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763608178580-73e2d97c-4b89-4049-bc0d-a5d3eaef5486.png)

7. 在“SMTP发件配置”和“发件模板配置”中：
+ 填写用于发送回执的 SMTP服务器、端口号、授权码（通常情况下，授权码可与收件授权码一致，可不填写；假如单独申请了发件授权码，可填写）；
+ 设置回执邮件主题和正文模板，支持配置以下三个汇总字段，按[&&]格式进行设置。收取文件数量：[&count&] 、处理成功发票数量：[&success_count&] 、解析失败文件数量：[&failed_count&]。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763608423883-a564ce35-61d0-40c7-afbe-26b9e76808a1.png)

+ 根据需要开启【发送处理结果详情附件】，开启后，将邮件文件详情列表以Excel文件形式作为附件发送。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763608431513-66adfdee-d8d6-4416-b8b0-dc24e72b8862.png)

8. 确认配置无误后，点击页面左上角【保存】。保存成功后配置自动启用。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763608443605-06d47883-fb67-4871-bc02-3b2f78054cdc.png)

**结果说明**：

+ 保存成功后，系统将自动同步邮箱配置到AWS，AWS开始定时提取该邮箱邮件。若需系统提取AWS取票的结果，需配置定时任务或手工发起取票，详见3.3.3节（发起邮箱取票任务（自动取票 + 手工取票））。
+ AWS定时邮箱取票，AWS的处理流程如下：
    - 根据邮箱配置登录需取票的邮箱。
    - <font style="color:rgb(0,0,0);">解析文件并做发票识别查验。（①支持提取的文件格式：zip、PDF、OFD、XML、图片、excel；②单个邮件大小限制为50M）</font>
    - <font style="color:rgb(0,0,0);">返回取票结果。</font>

#### 发起邮箱取票任务（自动取票 + 手工取票）  
**操作目标**：  
通过系统自动任务或手工立即触发，从收票邮箱中提取供应商发票。

**手工取票操作步骤**：

1. 确保至少已有一条“使用状态为可用”的供应商邮箱配置。
2. 点击顶部按钮【启动邮箱取票】，系统会根据配置的收票邮箱在后台创建取票任务。同一个收件邮箱只能同时执行一条任务，如已存在正在执行中的任务，将提示：邮箱*【邮箱账号*已存在正在运行的取票任务，启动后将进入等待队列，是否确认启动？

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1763609625156-dff395af-be43-4024-b9fd-742975173308.png)

**自动取票操作步骤**：

1. 实现定时提取邮件，需要在【调度计划】-【发票云】-【收票管理】启用调度计划：<font style="color:rgb(0, 0, 0);">rim_GetPullEmailTaskt_SKDP_S、rim_GetMailFileTask_SKDP_S。</font>
2. <font style="color:rgb(0, 0, 0);">定时任务默认</font>1小时执行1次，第一次提取近1天的邮件，后续以上一次的结束时间为开始时间~当前时间作为查询范围。
3. 系统将根据定时任务的时间策略，定时执行邮箱取票。

**结果说明**：

+ 系统将自动或手工提取AWS取票的结果（邮件及邮件中的文件信息），记录所有任务、提取邮件、提取文件的结果，查看邮箱取票任务处理情况详见3.3.4节（查询取票结果）。
+ 查验成功的发票入库，数据来源为：**邮箱采集。**
