## 进项全票池功能设计理念概述
1. **功能概述**

以组织的维度展示企业下的发票，可以查询发票不同业务流程的业务状态，如发票报销状态、发票抵扣认证状态，关联单据、凭证信息，展示发票全生命周期数据。

2. **目标客户以及客户解决核心痛点问题**

（1）内部操作使用人员：财务部门的员工，尤其是负责发票录入、查询、报销和会计处理的岗位。解决的核心痛点是提高发票处理的效率，通过自动化工具减少人工操作错误，实现发票全生命周期的透明化管理。

（2）部门管理人员：财务部门的管理人员，如财务经理或会计主管。解决的核心痛点是如何有效监督发票相关业务流程，确保发票数据的一致性和准确性，同时提高部门的工作效率和响应速度。

（3）公司管理层：公司的高级管理层，如CFO或CEO。解决的核心痛点是如何通过集中管理发票数据来优化公司的财务策略，降低税务风险，提升决策支持的数据质量和速度。

### 进项全票池主要操作说明
全票池管理采集识别的发票数据，进入全票池默认查询全部发票，也可在左侧栏选择指定发票类型查看；可对全票池数据进行查看发票、添加报销信息、维护入账信息、纸票签收、导出发票、删除。

全票池默认查询查验成功的发票，如需查询查验失败或未查验的发票，则需配置；<font style="color:#000000;">配置路径：开发平台-发票服务-系统管理-云应用参数配置-参数配置单据，配置信息如下：</font>

</font><font style="color:#000000;">配置项类型：rim_config, 配置项key：rim_config_show_uncheck, 配置项值为1：查询未查验的发票，0：只查询查验成功的发票。</font>
2. <font style="color:#000000;">发票发起报销、应付等关联了上游业务单据，业务单据给赋值的核算组织则为发票的核算组织，如果多次采集发票未用，多次上传组织覆盖，如果是已用或者在用，多次采集组织不覆盖 ，关于核算组织能否被覆盖可以自行配置，在系统网站地址后加上=rim_config，进入网页后按需求选择核算组织覆盖或者不覆盖。</font>![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1722945756975-4fbc193a-ff12-4996-9835-73029cdc11f4.png)
3. 可以对全票池中的发票设置哪些票种不能被删除，在系统网站地址后加上=rim_config，进入网页后按需求配置不能删除的字段和值。

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1722992352177-2c709558-9a9a-48ce-9a9f-b0ee4c5e4451.png)

4. EXCEL引入发票是否走查验配置：开发平台搜：参数配置单据 配置项类型：rim_config, 配置项：rim_config_need_check, 配置项值：0或1。（值为0时，不走查验，值为1时，走查验）。
5. <font style="color:rgb(0,0,0);">进项全票池，功能按钮权限</font>按列表上”核算组织“验权，不按用户登录组织；根据系统参数-基础服务参数：“隐藏无权按钮”，如果参数开了，无权的时候隐藏按钮；如果参数没开，无权也显示按钮，操作时出现提示语提示没权限。![](https://cdn.nlark.com/yuque/0/2025/png/47254879/1754531579135-750d85bd-dc47-410d-9ae2-680fbf93dd23.png)
