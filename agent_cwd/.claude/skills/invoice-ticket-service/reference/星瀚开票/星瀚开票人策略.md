### <font style="color:#333333;">应用场景</font>
<font style="color:#000000;">客户在发生开票业务之前，可以通过维护开票人策略，配置开票人、收款人、复核人。</font>**<font style="color:#333333;">我们所说的开票人匹配方案指的是匹配单据上的开票人字段，将被组装成开票数据用于开票</font>**<font style="color:#333333;">；</font>

### <font style="color:#000000;">开票人策略字段/按钮说明 </font>
| **<font style="color:#000000;">字段名称</font>** | **<font style="color:#000000;">详细解释</font>** |
| :--- | :--- |
| <font style="color:#000000;">开票人</font> | <font style="color:#000000;">开票人又称发票人。票据的当事人。即在各种有价证券(主要是本票、汇票和支票)上签名发行证券的人。发票人可以是个人，也可以是银行、商店及公司企业</font><font style="color:#000000;">；</font> |
| <font style="color:#000000;">收款人</font> | <font style="color:#000000;">收款人(PAYEE)收款人是指收取票款的人</font> |
| <font style="color:#000000;">复核人</font> | <font style="color:#000000;">复核人就是对发票开具的内容有检查和审核的人</font> |
| <font style="color:#000000;">当前登陆用户</font> | <font style="color:#000000;">取当前登陆用户名称</font> |
| <font style="color:#000000;">业务系统传参</font> | <font style="color:#000000;">取业务系统传参数</font> |
| <font style="color:#000000;">指定开票人</font> | <font style="color:#000000;">手动指定开票人，F7取系统基础人员</font> |
| <font style="color:#000000;">开票设备默认值</font> | <font style="color:#000000;">开票设备设定的开票人信息</font> |
| <font style="color:#333333;">手工录入</font> | <font style="color:#000000;">手工录入开票人信息</font> |
| <font style="color:#333333;">指定开票人</font> | <font style="color:#000000;">指定开票人信息，取系统人员基础资料</font> |


### <font style="color:#333333;">开票人策略业务校验</font>
<font style="color:#000000;">1、当设置了俩个开票人策略时，按照优先级排序优先匹配。</font>

2、 <font style="color:#000000;">用户可以配置多套开票人方案。并按照自定义条件来匹配方案。如有特殊业务需求，可以按照单据类型、组织等条件设置自定义条件</font>

3、 <font style="color:#000000;">开票申请单生成（botp推单、excel导入、手工新增、api接口推单）、待开发票、红字确认单都会分别匹配开票人策略</font>

4、 <font style="color:#000000;">没有配置策略，就取当前登录人，取不到就为空</font>

| **<font style="color:#333333;">业务对象</font>** | **业务场景** | **<font style="color:#333333;">是否匹配开票人策略</font>** | **<font style="color:#333333;">匹配逻辑</font>** |

| | <font style="color:#333333;">手工新增</font> | <font style="color:#333333;">匹配</font> | <font style="color:#333333;">1、取开票人配置</font><br/><font style="color:#333333;">2、没有配置取当前登录用户</font> |
| | <font style="color:#333333;">botp下推</font> | <font style="color:#333333;">匹配</font> | <font style="color:#333333;">1、取开票人配置</font><br/><font style="color:#333333;">2、没有配置取当前登录用户</font> |
| | <font style="color:#333333;">开票申请单接口</font> | <font style="color:#333333;">匹配</font> | <font style="color:#333333;">1、没有配置取当前登录用户</font><br/>2、 <font style="color:#333333;">取不到当前登录用户为空</font> |
| <font style="color:#333333;">扫码开票</font> | <font style="color:#333333;">静态二维码</font> | <font style="color:#333333;">不匹配</font> | <font style="color:#333333;">1、功能仅提交抬头</font> |
| | <font style="color:#333333;">动态二维码</font> | <font style="color:#333333;">匹配</font> | <font style="color:#333333;">1、没有配置取当前登录用户</font><br/>2、 <font style="color:#333333;">取不到当前登录用户为空</font> |
| | <font style="color:#333333;">小票码</font> | <font style="color:#333333;">不匹配</font> | <font style="color:#333333;">1、以填入的开票人为准，小票码要求必填</font> |

| | <font style="color:#333333;">excel导入待开</font> | <font style="color:#333333;">不匹配</font> | <font style="color:#333333;">1、以导入的开票人为准，excel要求必填</font> |
| | <font style="color:#333333;">单张开票</font> | <font style="color:#333333;">匹配</font> | <font style="color:#333333;">1、取开票人配置</font><br/><font style="color:#333333;">2、没有配置取当前登录用户</font> |
| <font style="color:#333333;">发票红冲</font> | <font style="color:#333333;">发票红冲</font> | <font style="color:#333333;">不匹配</font> | <font style="color:#333333;">1、取原蓝票开票人</font> |
| | <font style="color:#333333;">红字信息表</font> | <font style="color:#333333;">匹配</font> | 1、 <font style="color:#333333;">红字信息表开具时匹配开票</font><br/>2、 <font style="color:#333333;">没有配置取当前登录用户</font> |
| | <font style="color:#333333;">红字确认单</font> | <font style="color:#333333;">不匹配</font> | <font style="color:#333333;">1、取原蓝票的开票人，支持手工编辑</font> |


### <font style="color:#000000;">主要操作</font>
<font style="color:#000000;">1)</font><font style="color:#000000;">   </font><font style="color:#000000;">点击【新增】开票人策略，包含【开票人】、【收款人】、【复核人】三个字段的配置。点击新增，按照提示填充开票人设置基础信息。包含策略名称、适用组织（支持多选）、优先级。当设置了俩个策略同时满足时，按照优先级排序优先匹配。</font>

<font style="color:#000000;">2)</font><font style="color:#000000;">   </font><font style="color:#000000;">可以自定义匹配条件为该策略的启动条件。如有特殊业务需求，可以按照单据类型、组织等条件设置自定义条件。</font>
