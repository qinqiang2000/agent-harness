## 拆分配置功能设计理念概述
<font style="color:#000000;background-color:#FFFFFF;">拆分和合并功能是开票过程中的核心模块。拆分功能用于对单据或明细进行处理，例如将一张开票申请单拆分为两张发票。用户在使用时，可以将此功能与合并配置和拆合方案配置结合使用。</font>

<font style="color:#000000;background-color:#FFFFFF;">在企业进行开票业务时，若单据金额超过限额或需要按相同税率分别开票等特定需求，可以通过发票拆合规则进行配置。用户可根据自身的业务需求定义多种拆合规则。在开票时，只需指定相应的合并或拆分规则编号，即可按照所选规则进行有效的拆合处理。</font>

### 拆分配置主要操作说明
##### 新增拆分设置
**步骤1：**点击“新增”按钮，进入拆分设置配置页。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748594828318-0aeae7ef-0f56-4e51-8f54-c12b584682f4.png)

  
**步骤2：**配置“基本信息”。

注：

+ <font style="color:#000000;">同一机构下，规则编号不允许重复。</font>
+ <font style="color:#000000;">系统默认规则不允许删除、禁用、修改名称。</font>
+ <font style="color:#000000;">自定义规则允许修改、禁用，禁用后，再使用该规则编号，则不生效，系统默认按“默认规则”处理。</font>
+ <font style="color:#000000;">规则编号不得修改，对于不再使用的规则可进行删除。</font>

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748594866786-ed55ee45-958a-49ab-b816-801942ea1b85.png)  


**步骤3：**配置单据拆分规则。

**单据拆分规则：**主要场景针对开票申请单按某个限额，拆分成多张的规则设置。

1. **按指定不含税金额拆分：**按开票申请单类型，根据设置的固定金额拆分成多张发票，如一张纸质专票设置限额为1万，开票申请单3万元，若不考虑明细拆分，系统将拆分成3张1万的发票开具。
2. **不含税金额超过税盘票种限额：**该选项不可修改，主要旨在提醒，设置的限额不能超过税盘的限额，如税盘限制开票最大限额为3万元，但上方设置了4万元，这种设置会导致最终开票失败。 

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748594892861-3b946da0-6d04-40bf-8622-335bfbfaaca1.png)  


**步骤4：**配置商品行拆分规则。

**商品行拆分规则：**主要场景针对开票申请单对于明细的拆分开票规则。

1. **保留小数位设置：**涉及某行明细需拆分时，数量、单价保留指定的小数位。
2. **数量位数处理：**注意，此处说的是数量的小数点后的位数，在拆分的过程中位保障税额误差符合税局的要求，可能会有除不尽情况，此参数即为设置当计算拆分数量位数大于设置的小数位数时，系统的逻辑，如数量设置2位，但按实际计算后，数量应显示未1.23456才能计算出正确的税额。<font style="color:#000000;">此处有两种设置如下：</font>

<font style="color:#000000;">①单行商品税额误差超过范围时，以系统为准，即显示1.23456。</font>

<font style="color:#000000;">②单行商品税额误差超过范围时，强制保留数量位数，反算不含税单价，即数量显示1.23，但系统会通过1.23这个数量去结合金额倒算单价</font>**<font style="color:#000000;">。</font>**<font style="color:#000000;">	       </font>

3. **数量设置：**此处针对某类特殊场景，即对应明细无论拆分几张发票，数量都填写固定值，此处适应于如餐饮或服务类发票。
4. **<font style="color:#000000;">商品行拆分规则</font>****：**业务场景为满足商品单价、数量指定小数位，被拆分商品在最后一张发票上可能会出现：金额≠数量*单价，为保证被拆分商品的总金额恒等，系统将调整被拆分商品的数量、单价、金额，不同设置将影响最后一张发票中单价、数量，同一单据因超限被拆分多张，且加上发票最后一行商品金额正好超限额，此时是否需要将最后一行商品行按数量拆分到多张发票中。系统有三种处理办法如下：

①不拆分，即最后一张发票若超限，则按实际开具，不再匹配拆分条件。

②调整数量，保证单价不变：最后一张拆分数量=剩余金额/单价，可能最终总数量不等，数量保留小数位设 置有效。

③调整单价，保障数量不变：最后一张单价=剩余金额/剩余数量，可能最后一张发票上商品单价不一致，单价保留小数位设置有效。

详见下方举例逻辑：

![](https://cdn.nlark.com/yuque/0/2023/png/26098688/1685771928815-cebce95f-7b2e-41ea-a2d4-0dfa61a773fe.png?x-oss-process=image%2Fformat%2Cwebp%2Fresize%2Cw_1231%2Climit_0)

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748594924074-5bedfe4f-25a5-431d-88d4-ccd5123e8b67.png)

  
**步骤5：**配置其他规则。

**其他规则：**设置拆分中对于备注生成、清单生成规则，清单行数及税额计算规则。

1. **发票备注规则：**假设某张单据的单据头备注为“Z”该单据有三条明细，每条明细的备注为A B C ，处理规则如下：

①只取单据备注，发票无论拆分几张备注为“Z”。

②单据备注+单据第一行明细备注，如ABC各生成一张发票，则三张发票的备注为Z+A、Z+B、Z+C。

③单据备注+叠加**<font style="color:#DF2A3F;">发票</font>**明细备注，如AB生成一张发票 C生成一张发票，则一张显示Z+A+B，一张显示Z+C。

④单据备注+叠加**<font style="color:#DF2A3F;">单据</font>**明细备注，如AB生成一张发票 C生成一张发票，则所有的发票上都显示备注     Z+A+B+C。

2. **清单处理规则（数电后无清单概念）**：

①不生成清单，商品行超过发票最大允许行，新增一张发票，按一张发票最大支持8行的标准，超出后生成新的发票。

②商品行超发票最多允许行数时，开具成清单发票，如超出8行，则生成一张发票+清单③不管商品行是否超发票最多允许行数，强制开具成清单发票，无论发票明细多少行都生成清单。

3. **发票最多行数：**设置发票/发票清单可开具的最多行数。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748594951043-f06788e4-fe80-46cb-b749-4b6c05192406.png)  


**步骤6：**拆分字段配置。

可按照自定义字段对发票进行拆分。例如按税率进行拆分，相同税率生成一张发票。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748595001227-5ad3f3b3-f760-4c04-8ae5-13cc739e9eee.png)
