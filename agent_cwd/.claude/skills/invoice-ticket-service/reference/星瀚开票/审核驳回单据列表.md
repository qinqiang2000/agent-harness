## 审核驳回单据列表功能设计理念概述
审核驳回单据列表主要用于集中展示业务系统/ERP审核单据过程中被驳回/退扫的影像单据，便于统一查看、定位退扫原因并发起重扫。

**<font style="color:rgb(0, 0, 0);">用户痛点：</font>**

1. <font style="color:rgba(0, 0, 0, 0.85) !important;">传统审核驳回后，影像文件需线下重新扫描上传，流程耗时长、易出错。</font>
2. <font style="color:rgba(0, 0, 0, 0.85) !important;">ERP 审核结果与影像系统数据不同步，需人工确认退扫需求，操作割裂。</font>
3. <font style="color:rgba(0, 0, 0, 0.85) !important;">单单据逐一处理退扫，缺乏批量操作能力，面对大量驳回单据时效率低下。</font>
4. <font style="color:rgba(0, 0, 0, 0.85) !important;">无法直接在系统内修改驳回影像，需重新采集全部文件，浪费资源。</font>

**产品价值：**

1. <font style="color:rgba(0, 0, 0, 0.85) !important;">通过线上化重扫流程，将传统 “线下扫描→人工上传→系统提交” 的多步操作简化为 “系统内修改→一键提交”，耗时缩短 70% 以上。</font>
2. <font style="color:rgba(0, 0, 0, 0.85) !important;">与 ERP 深度联动，确保审核结果与影像状态实时同步，避免因信息滞后导致的重复驳回。</font>
3. <font style="color:rgba(0, 0, 0, 0.85) !important;">支持单单据快速修改重扫，减少重复劳动；操作日志可追溯，降低人为错误排查成本。</font>

### 3.3 审核驳回单据列表主要操作说明
##### 审核过程中被退扫影像的重扫流程 
<font style="color:#000000;">审核人在ERP中对单据进行“退扫”或者“驳回并退扫”时，影像文件需要在影像系统重传，此时可以使用影像重扫功能。</font>

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1747298104933-45f09b20-3fd4-49f6-9600-030cf331a0df.png)

**<font style="color:#000000;">步骤1</font>**<font style="color:#000000;">：在“审核驳回单据列表”点击“影像单据编号”蓝色链接，进入影像重扫界面。</font>

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1747296506701-2de46855-fafa-416f-a3c1-2e259aeeb52f.png)

**<font style="color:#000000;">步骤2</font>**：在影像重扫界面修改影像文件，并点击“提交”按钮重新上传。提交之后可以继续在业务单据进行下一步操作，例如审核。

影像重扫页面具体可进行的操作见此链接3.3节：[https://jdpiaozone.yuque.com/nbklz3/grvkab/beimwsd7u8dqp5bb](https://jdpiaozone.yuque.com/nbklz3/grvkab/beimwsd7u8dqp5bb)
