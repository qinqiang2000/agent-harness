## 静态二维码开票功能设计理念概述
### 功能概述
C端用户扫描固定桌牌二维码填写购方抬头信息和收票方式等信息后提交开票请求，开票请求统一汇入开票列表待审核后开票.

应用场景： C端用户扫描固定桌牌二维码扫码后，填写开票的公司名称、税号、地址电话、银行及账号等信息，提交到扫码开票列表，B端商家根据提交的信息再进行开具发票。对零售、餐饮、酒店等企业来说这样开票不仅减少核对客户信息的时间，也不容易出错。提升开票效率。

### <font style="color:rgba(0, 0, 0, 0.85);">目标客户以及客户解决核心痛点问题</font>
1. <font style="color:rgba(0, 0, 0, 0.85);">企业外部消费者/外部客户：需要快速、便捷地获取发票的消费者或客户，解决了在传统开票方式中，需要等待较长时间或提供复杂信息的痛点，通过扫描静态二维码即可快速开票。</font>
2. <font style="color:rgba(0, 0, 0, 0.85);">企业内部操作使用人员：财务或相关开票岗位，解决了手动输入开票信息的繁琐问题，提高了开票效率，减少了错误率。</font>
3. <font style="color:rgba(0, 0, 0, 0.85);">部门管理人员：财务部门管理人员，解决了对开票流程进行监控和管理的痛点，能够确保开票操作的合规性和准确性。</font>
4. <font style="color:rgba(0, 0, 0, 0.85);">公司管理层：公司高层管理岗位，解决了对公司开票情况进行整体把握和决策的痛点，能够通过静态二维码开票的数据统计，了解公司的业务情况和客户需求，为公司的发展提供决策支持。</font>

###  静态二维码开票主要操作说明
#### 静态二维码开票
**步骤1：**选中要操作的单据，点击”扫码开票“按钮，进入发票编辑界面。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748584124672-9dfc4774-c142-4b27-aaca-949de4a9f05e.png)

**步骤2：**完善开票基本信息。填写包括此次要操作的组织、“发票类型”和“征税方式”等，由创建者根据实际情况和需要进行设定。选择票种，不同特殊票种有不同的填写项目，请根据实际要求填写发票信息。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748584143831-177fbe4a-bf38-4570-b41c-158096e123dd.png)

**步骤3：**完善购方抬头信息。在名称的下拉菜单中选择，税号、地址等信息可以自动匹配出。在基础资料基础信息设置中开启购方抬头的自动保存功能，可以将单张开票使用过的购方抬头信息自动保存起来，方便后续快速选择开票。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748584164384-566bbafe-8a45-4ba0-b0cf-a5968cd6f16f.png)

**步骤4：**完善项目信息。点击“增加明细”按钮，添加新的项目。在项目名称的框内按ENTER键即可在菜单中选择已有商品，填写规格型号、单位、单据等信息。确定税率后，税额可由系统计算出。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748584179461-110a6f22-6b43-4355-ad01-e3940865b494.png)

**步骤5：**填写备注、收款人、复核人的其他销售方信息。

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1722306092756-d442ed6e-0a30-4762-9af9-b9e7aa0bbdfb.png)

**步骤6：**点击“更多”按钮，还可填写购方经办人信息。

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1722306281718-fdfacb01-4c5f-493f-b636-19b0ddeaaa7d.png)

**步骤7：**点击“开票”按钮。

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1722306429400-77bb750a-ec24-413a-892d-7ff8ce989c45.png)
