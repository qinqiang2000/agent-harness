## 对账汇总开票功能设计理念概述
 对账汇总功能分为单据对账和金税对账。单据对账是提供给用户进行开票申请单开票金额和已开发票金额的对比功能；金税对账是提供给用户进行发票云已开发票的数据与金税数据的比对功能；旨在发票开票数据中的差异。

### 用户痛点
1. 各业务数据开具发票后，由于明细合并等操作，可能会造成单据金额与发票金额不一致的情况。
2. 由于网络等原因，发票云与税局交互异常，导致发票信息出现偏差，发票云开具的发票与税局实际开出发票核对复杂。

### 解决客户问题的方案
通过单据金额和对应开票金额、发票云发票数据与税局发票数据的统计报表，帮助企业快速核对发票数据。共包含三个报表：

（1）单据对账：用于核对单据与单据最终开票金额的差异

（2）金税对账：用于核对发票云的开票金额、数量等信息与税控设备的差异，仅支持金税盘

（3）数电对账：用于核对发票云的开票金额、数量等信息与电子税局的差异

### 对账汇总主要操作说明
#### **单据对账查询**
**步骤1：**设置过滤条件，即可看到该条件下的对账汇总详情。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748587078495-16f1e69f-843b-4a85-9aa9-7edfb4706415.png)

**步骤2：**点击“导出”按钮，将数据已EXCEL表的形式引出。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748587094370-0e20dc2e-b05f-42b5-bb85-edea07431fdf.png)

#### **金税对账查询**
**步骤1：**设置过滤条件，点击“查询”按钮，即可看到该条件下的对账汇总详情。暂不支持百望盘，税务UKey对账。组织选择设备不在线时也无法查询。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748587112209-d7f03354-b504-4ec5-877f-553c00f6fc47.png)

**步骤2：**点击“导出EXCEL”按钮，将数据已EXCEL表的形式引出。

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748587126617-4574708e-41ec-4824-b6a5-85b859a19f6e.png)

#### **数电对账查询**
**步骤1：**<font style="color:rgb(0, 0, 0);">选择组织，将按该组织所属税号对比。设置过滤条件，点击“查询”按钮，即可根据筛选条件查询税局，获取数据后，根据发票号码、购方税号、开票日期、价税合计、合计税额进行匹配对账（数电票个人税号为空，个人时比对逻辑不校验税号），均一致则认为对账成功，不一致时视为对账失败，对账失败情况包括：  
</font><font style="color:rgb(0, 0, 0);">①根据以上匹配条件对账失败，展示在发票云的发票数据，对账失败原因提示：该发票在发票云和税局明细不一致。  
</font><font style="color:rgb(0, 0, 0);">②发票云存在发票数据，税局不存在，发票数据展示发票云数据，对账失败原因提示为：该发票在税局不存在。  
</font><font style="color:rgb(0, 0, 0);">③发票云不存在该发票，税局存在，发票数据展示税局发票数据，对账失败原因提示为：该发票在发票云不存在。</font>

<font style="color:rgb(0, 0, 0);">点击发票号码，可查看发票文件。</font>
