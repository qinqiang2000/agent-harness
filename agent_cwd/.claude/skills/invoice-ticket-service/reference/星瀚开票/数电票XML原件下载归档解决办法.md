## 政策背景：
1、5月20日，财政部会计司关于公布电子凭证会计数据标准通知，数电票需要有数字签名的XML文件进行报销入账归档，可不再另以纸质形式保存。接收方如果以数电PDF/OFD格式文件的纸质打印作为<font style="color:#DF2A3F;">报销入账归档</font>依据的，应该同时保存含数字前面的XML格式电子文件。

![](https://cdn.nlark.com/yuque/0/2023/png/647520/1702343208038-9bf00eeb-073e-4525-aea7-6467ea7b8d31.png)

2、11月底，乐企将取消数电票原件生成下载能力，数字化电子发票版式及 XML 规范（基础版开票）已经下发至企业，企业可根据数字化电子发票版式及 XML 规范自行生成 XML版式（无需数字签名）

[数字化电子发票版式及XML规范V1.pdf](https://jdpiaozone.yuque.com/attachments/yuque/0/2023/pdf/647520/1702343391786-150e9bf6-8e07-4d96-8c1f-5eb896faba96.pdf)



## 现状问题：
1、税局与财政部政策本身存在冲突。XML数据格式本身不利于普通用户使用和浏览（结构化数据）；

2、企业及发票服务商获取数电票XML原件较为困难，目前仅电子税局提供单张数电票的XML原件下载，批量收票下载存在较大性能问题（税局服务成本较大）。



## 解决方案：
** 1、若为电子税局通道：**在【全量发票数据同步设置】，配置下载类型为“进项-全量”，启用金税四期下载数电发票版式文件开关。即可从电子税局下载数电票版式文件，支持格式包括 PDF、OFD 和 XML。

**2、若为乐企通道：**在【全量发票数据同步设置】，全量发票数据同步下载类型为“进项-全量”或”进项-增量“，启用金税四期下载数电发票版式文件开关。可用定时任务调用RPA接口下载PDF、OFD和XML（乐企企业下载版式文件免配置RPA电子税局账号）。
