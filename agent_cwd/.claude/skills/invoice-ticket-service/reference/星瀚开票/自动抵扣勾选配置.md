## 自动抵扣勾选配置功能设计理念概述
<font style="color:#303030;">部分企业手工勾选工作量大，可以通过设置”自动抵扣勾选“实现进项发票勾选的自动化。（此配置只自动做勾选，不自动申请统计，确认签名）</font>

<font style="color:rgb(48, 48, 48);">通过配置自动抵扣规则，系统定时任务会针对符合配置中条件的发票进行自动勾选。</font>

### 自动抵扣勾选配置主要操作说明
#### <font style="color:#303030;"> 配置自动抵扣勾选</font>
<font style="color:#DF2A3F;">如果上游业务单据已经标识发票是做抵扣勾选、不抵扣勾选、退税勾选。则执行任务时，以上游系统的为准，如果没有标识则默认为抵扣勾选。</font>

<font style="color:#DF2A3F;">如果核算组织在发票服务基础资料配置的收票通道是“新电子发票服务平台”请确保在自动勾选调度计划“rim_AutoDeductSelectTask_SKDP_S”的调度时间内保持收票默认电子税局账号的登录。</font>

<font style="color:#DF2A3F;">手工执行撤销勾选的发票系统在自动勾选中不会勾选该发票。</font>

<font style="color:#DF2A3F;">记录自动抵扣勾选的次数，6次失败（RPA或乐企返回失败），则不再重试自动抵扣，需要手工操作勾选。</font>

**步骤1：**新增自动抵扣配置列表

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1721371085854-429315f8-7ae1-4d24-8f61-550ad39dc450.png)

**步骤2：**编辑基础信息

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748593740857-fdea55d7-13f7-4d33-8307-a77916da441d.png)

**步骤3：**选择适用组织

点击“增行”后，按照下图箭头指示可以选择使用组织（选择使用组织是为了分配这条配置给某个企业使用），系统会根据选择的组织信息自动填入纳税人税号和纳税人名称。适用组织是用来规定此条配置对哪些组织生效，此处与自动抵扣勾选配置列表中的控制策略区分开来，控制策略是规定哪些组织有维护管理该条配置的权限。

<font style="color:rgb(0, 0, 0);">添加组织或是查看已经创建好的组织时，系统将查询方案里的全部核算组织的数电通道，若通道为“新电子发票服务平台”，则查询数电配置中是否有默认业务为收票的电子税局账号，无账号提示获取当前属期失败：税号：[XXXXXX]未查询到账号信息，请前往基础资料-企业管理-数电配置进行相关配置。</font>

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748593754952-4e7757b7-691f-40b4-a30b-c1c955b7792c.png)  
**步骤4：**数据范围选择

**数据范围**：因进项全票池的来源非常多，所以需要根据业务情况选择通过什么渠道来源的发票、不同的发票种类、指定发票状态、指定风险等级、指定税局入账状态、指定的业务单据来源的发票可进行自动勾选抵扣，根据实际使用要求配置即可。

“发票状态”支持选择正常、部分红冲（税局仅允许勾选正常和部分红冲的发票）。

“风险等级”支持选择正常、疑点发票（①税局禁止异常凭证勾选，疑点凭证可提交勾选成功，但可能产生税务风险，请慎选；②在系统中，有部分未同步到风险等级的发票为无，对此类发票进行勾选时，系统会传正常到税局勾选----若税局勾选成功，系统会根据税局返回的风险等级更新票池数据）

“按业务单据”+“单据抵扣标识”：在“按业务单据”选择了单据类型，根据发票关联的单据传的是否可抵扣，单据传是否可抵扣=是，则对应单据抵扣标识=是。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1761015597000-10f75846-fe52-406a-8a4e-8a879bc08923.png)

**步骤5：**业务状态选择

**业务状态**：按不同的业务状态来进行自动勾选，如财务要求必须已签收的发票才能勾选，则设置按签收状态选择“已签收”。

（签收状态、是否可抵扣、预勾选等字段含义可在[进项全票池](https://jdpiaozone.yuque.com/nbklz3/tadboa/ok9tcmo0a794yd90)中搜索。）

按是否可抵扣：发票服务会根据票面信息判断是否符合税局可抵扣要求；单据使用发票时，单据又可根据实际业务情况进行修改，以单据的为准。

按查验状态：对<font style="color:rgb(0,0,0);">增值税代扣代缴完税凭证不生效。</font>

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1761017120389-fe820ce7-8ec0-41b7-8ca8-f16679394c53.png)

**步骤6：**审计状态选择

**审计状态**：<font style="color:rgba(0, 0, 0, 0.85);">按不同的审计状态（发票预警结果）来进行自动勾选，用户的客户非常少，建议设置不限。</font>

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1721388538589-397b4fea-96ca-4e5f-a4fe-4459b53a8181.png)

**步骤7：**设置抵扣规则

抵扣规则**：**

+ 按固定税负率倒算进项税，进行抵扣。
+ 按可抵扣票据全自动抵扣。

以上两种抵扣规则选择其一即可。

1）设置抵扣规则-->按固定税负率倒算进项税，进行抵扣

**（**<font style="color:#000000;">理论小知识：销项-进项，实际算出来的税，是企业的税负率，即一个企业实际上每期交的税跟自己销售额的比例，大部分企业的税负率都是稳定在一个相对固定的数值附近，税负率大幅变化都会引起税局的关注，故企业在进项充足的情况下都会控制这个比例，预留进项到下期，如进项不够，一般企业会采取控制销售的方式来控制税负</font>**）**<font style="color:rgba(0, 0, 0, 0.85);">这种方式下，会按照税负率来整体控制勾选抵扣的范围及发票。</font><font style="color:#DF2A3F;">按固定税负率倒算进项税，由于每张发票金额不同，实际抵扣值小于或等于计算值。</font><font style="color:rgba(0, 0, 0, 0.85);">具体数据需根据具体政策条件自己手动填写。</font>

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1721387972373-3f6cd6f0-b907-4eaa-899b-ba4eb9d2f21c.png)

取数方式有两种：按采集时间先进先抵、按开票时间先进先抵。

+ 按采集时间先进先抵：<font style="color:rgba(0, 0, 0, 0.85);">按照进项发票或其他抵扣凭证被采集到系统中的时间顺序，先采集的先进行抵扣</font>
+ 按开票时间先进先抵：<font style="color:rgba(0, 0, 0, 0.85);">依据发票的开票日期，先开具的发票对应的进项税额先进行抵扣</font>

<font style="color:rgba(0, 0, 0, 0.85);">此两种取数方式需根据自己的需要进行选择。</font>

2)  设置抵扣规则-->按可抵扣票据

![](https://cdn.nlark.com/yuque/0/2025/png/56650036/1748593772794-04edb24f-c3c5-4d83-a4e0-2a0015a7ce43.png)

配置完成，点击保存。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1745245386125-bf5ba11c-0807-4403-9a55-feadbd16e8e6.png)

在列表处，可以对方案进行启用或禁用。

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1745245446892-8a916491-eca2-42a8-aa64-041668de871d.png)

注意：<font style="color:rgb(0,0,0);background-color:rgb(255,255,255);">用户在星空做抵扣类操作（</font><font style="color:#8A8F8D;background-color:rgb(255,255,255);">发票抵扣勾选、发票不抵扣勾选、海关缴款书抵扣勾选、海关缴款书不抵扣勾选、增值税代扣代缴完税凭证抵扣勾选、增值税代扣代缴完税凭证不抵扣勾选、撤销勾选查询统计表、申请统计表、撤销统计表、确认统计表、农产品发票类型上传确认，发票退税勾选、海关缴款书退税勾选、退税勾选统计、变更税款所属期、</font><font style="background-color:rgb(255,255,255);">自动抵扣勾选配置保存方案、自动抵扣勾选配置启用方案、</font><font style="color:#8A8F8D;background-color:rgb(255,255,255);">批量抵扣方案保存方案</font><font style="color:rgb(0,0,0);background-color:rgb(255,255,255);">）时，若判断为小规模纳税人，则阻断提交任务到税局通道，提示：</font><font style="color:rgb(0,0,0);">小规模纳税人不支持抵扣类勾选，可根据企业在税局的信息，在星空发票服务企业基础资料修改。</font>

**执行自动勾选时，产品勾选逻辑如下：**

用户在星空配置自动勾选，或是税局用票日志列表点击处理失败的抵扣类勾选任务重试，重新执行以下全部步骤。

    1. 发票必须是未勾选或预勾选状态，否则不能提交税局勾选。
    2. 读取税号的企业基础资料的税局收票通道类型，为乐企或电子税局进行一下步骤，否则保持现有逻辑不变。
    3. 进行需要做发票信息和统计表的校验。
        1. 是否小规模纳税人：根据发票的购方税号，读取星空发票服务-基础资料-企业信息的企业性质，如果是小规模纳税人则勾选批次为处理失败，失败描述：小规模纳税人不支持抵扣类勾选，可根据企业在税局的信息，在星空发票服务企业基础资料修改。
        2. 是否旅客运输抵扣，旅客运输抵扣跳过下述校验直接按现有的逻辑勾选。
        3. 是否可勾选的发票类型：除以下发票类型，勾选批次为处理失败，失败描述：包含不可勾选的发票类型。

<font style="color:#585A5A;">电子专票</font>

<font style="color:#585A5A;">数电票（增值税专用发票）</font>

<font style="color:#585A5A;">纸质专票</font>

<font style="color:#585A5A;">机动车</font>

<font style="color:#585A5A;">通行费</font>

<font style="color:#585A5A;">海关缴款书</font>

<font style="color:#585A5A;">数电票（航空运输电子客票）</font>

<font style="color:#585A5A;">数电票（铁路电子客票）</font>

<font style="color:#585A5A;">数电发票（机动车销售统一发票）</font>

<font style="color:#585A5A;">增值税代扣代缴完税凭证</font>

        4. 统计表是否已生成：星空缓存的税号当期统计表状态为已生成，则勾选批次为处理失败，失败描述：当前税款所属期统计表已生成，请撤销统计再勾选，或使用【当期抵扣勾选统计】查询更新统计。
    4. 校验通过，检测登录。
    5. 未登录且是人工登录的账号弹窗提示登录，不登录跳过统计表查询和统计表状态校验。
    6. 登录成功或自动登录的，查询企业统计表。
    7. 查询统计表成功，星空缓存的税号当期统计表状态30分钟。
    8. 校验统计表是否已生成：星空缓存的税号当期统计表状态为已生成，则勾选批次为处理失败，失败描述：当前税款所属期统计表已生成，请撤销统计再勾选，或使用【当期抵扣勾选统计】查询更新统计。
    9. 提交乐企或RPA通道勾选。

勾选任务失败的，在【税局用票日志列表】选择状态“处理失败”的任务重试，需要重复以上流程的校验。

#### <font style="color:#303030;">  系统自动抵扣任务执行和结果查询</font>
<font style="color:#303030;">系统每日定时任务执行，针对所有可用状态的自动配置规则进行勾选操作。勾选任务执行结果查看路径在【发票服务】-->【收票管理】-->【进项抵扣】-->【税局用票操作日志列表】，以列表形式预览，可查看发票勾选日志，点击批次号链接即可查看该批次下具体发票勾选详情。</font>

![](https://cdn.nlark.com/yuque/0/2023/png/25809591/1691652836189-eaf56d0c-e19d-4275-88c8-49561a188b07.png)

#### <font style="color:#303030;">手动执行自动抵扣任务</font>
如需手动执行自动抵扣任务，进入【系统服务云】-->【调度中心】-->【调度计划】-->【发票服务】，选择调度任务“自动勾选”（rim_AutoDeductSelectTask_SKDP_S）”手工执行“即可。
