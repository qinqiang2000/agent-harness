| **乐企停止提供XML下载服务后，如何获取XML？** | <font style="color:rgb(0, 0, 0);">乐企停止提供XML下载服务后，纳税人从开票方获得XML。开票方从电票平台开出的数电票，由电票平台生成XML并交付提供下载服务；开票方通过乐企直连模式在自有系统开出数电票的，由开票方自建平台按照税务机关标准生成XML并负责交付。税务机关对接入乐企的直连单位如何生成和交付数电票XML及版式文件有正式的规范，要求乐企接入单位开出数电票后立即通过自有渠道（包括自建APP、电子邮件、其他第三方票夹应用等）完成数电票XML及版式文件的及时交付，税务机关也将对乐企开票方是否按照规范及时生成并交付数电票XML及版式文件进行实时监督。</font> |
| 乐企开票是否有明细行限制？ | <font style="color:rgb(0, 0, 0);">由于电票平台目前的明细行限制在5000行以内，乐企开具的发票会推送到电票平台，所以不建议明细行过多，否则会导致pdf生成不成功、查询下载明细出错等问题。</font> |
| **乐企开票日开票上限是多少？** | <font style="color:rgb(0, 0, 0);">目前没有对单户企业数据上传量作出限制，现在可支持单户企业数据上传量达200万。（例如某大型企业一天上传数据量已达到800万）</font> |
| **海外总部企业如何申请乐企自用服务？** | <font style="color:rgb(0, 0, 0);">按照《乐企自用直连服务的规范指引》要求，乐企自用使用单位与直连单位为同一总分公司、集团企业或具备相互股权控制关系的的企业。可以由一家作为直连单位，其他单位作为使用单位接入时需出具与直连单位的关联关系证明。</font> |
| **全国性企业在乐企自用申请中的申请次序是怎样的？** | <font style="color:rgb(0, 0, 0);">按照《乐企自用直连服务的规范指引》要求，第一步，直连单位（如公司总部）当地主管税务机关申请，申请确认后即可接入开票；第二步，由直连单位向使用单位（如子公司）发送邀请信息；第三步，使用单位确认邀请信息并向使用单位主管税务机关发起使用请求，使用单位主管税务机关确认后，即可接入开票。</font> |

| 总省两级运维模式是什么？ | 按照“金四”运维总体要求，“金四”采取总省两级一体联合运维模式。   运维问题受理渠道主要包括电子税务局征纳互动平台、12366 热线、联络员提供的乐企服务相关操作辅导、咨询解答等征纳互动渠道，以及直连单位和使用单位的主管税务机关与相关纳税人建立的乐企平台运维沟通渠道。   主管税务机关负责安排专人建立并维护乐企平台运维沟通渠道，跟进乐企平台运行情况，推送乐企相关指引、变更、通知；及时收集并上报纳税人报送的系统相关问题。对于突发性阻断类问题，快速上报至省局运维团队。同时，应安排专人全程跟进上报问题的解决进度，并将结果及时反馈至乐企纳税人。定期完善征纳互动平台乐企服务知识库、推送指引文章等。   省局负责建立乐企运维服务团队，对收到的问题进行分析、定位、解答，无法处理的问题转交总局运维团队处理，对于突发性阻断类问题，快速上报至总局跟进处理。   总局项目组建立乐企运维团队，及时处理各省市运维团队上报的问题。 |
| 申请了乐企试点的纳税人可以退出吗？ | 坚持企业自愿原则，未要求申请了乐企试点的纳税人不得退出。同时，根据乐企直连规范要求“乐企自用的初次接入时间为 2 个   自然年度，需延续使用的，应在到期前 3 个月发起延期请求……到期仍未提交延续请求的，税务机关将终止平台乐企接入资格”。因此，申请了企业试点，2  个自然年度未完成初次接入且未提交延续请求的，税务机关将终止平台乐企接入资格。 |

| 一个集团企业可否有多个直连单位？早期试点时，集团企业是指定的集团内成员企业作为直连单位的，现集团企业总部可否作为使用单位？ | 一个集团可以有多个直连单位（接入单位）。按照总局直连服务规范指引，直连单位是指直连平台所有者，即主要责任单位，应为总分支机构的总公司、集团企业总部、具备股权控制关系的实际控制单位等。集团不是一家具体的公司或企业，而是依托股权或其他控制关系组成的一组企业集群。依据商业模式和组织架构的不同，一个大的企业集团可以由多个较小的企业集团组成，集团也可以有多个不同层级的总部。通常情况下，集团全球总部是组成该集团的所有公司或企业中的最终控股母公司（能够在财务上将集团内其他企业纳入自己的合并报表且无法再被集团内外其他企业纳入合并报表的企业），除全球总部之外的其他集团总部是组成该层级集团的一组公司或企业中的控股母公司。例如：A 集团全球总部，A 集团大中华区总部， A 集团华北区域总部等等。一个集团可以将其内部较小的集团的总部作为直连单位。 |
| 如何查看能力编码？ | 试点纳税人可以在乐企平台【能力中心】——＞【能力申请管理】——＞点击【详情】查看能力编码。 |
| 在哪里查看乐企使用单位ID？ | 包括两种方式。方式一：直连单位登陆乐企，从【准入管理】——＞【准入邀请】，在邀请使用单位列表中，选择【查看详情】，可查看使用单位的ID。   方式二：由使用单位登陆乐企，从【准入管理】——＞【准入申请】页面，查看使用单位的乐企ID。   备注：在企业请求报文头中，直连单位平台编号处填写直连单位的乐企ID，使用单位平台编号处填写使用单位的乐企 ID，表示使用单位和直连单位的身份。   在企业请求开票接口报文体中，平台编号处填写对应直连单位的乐企ID，表示该张票通过哪个直连单位的通道开具。 |
| 直连单位与使用单位的“平台编号（ptbh）”是分开的还是唯一的呢？ | 每个单位（不论直连单位，使用单位）的ptbh 是唯一且独立的。 |

| 发票上传接口中区域代码是否是必填项目？ | 发票上传接口中区域代码为必填项，目前没有码值列表，需填写销售方省税务局税务机关代码( 如广东省税务局机关代码： 14400000000)。若省级税务机关代码不清楚，可咨询对应省级税务机关。 |
| 可否提前提供接口URL 地址？ | 不能。试点纳税人按照流程操作就能获取到 URL 地址，该地址由系统根据流程及纳税人有关信息加工生产，提前预给的地址无法登录使用。 |
| nlbm、ylbm 在沙箱环境与生产环境中都需要填写吗？在沙箱环境与生产环境中的编码是否一致？ | 问题中提到的 nlbm 指的是能力编码，ylbm 指的是用例编码。在进行沙箱测试或者能力请求时，nlbm 在沙箱环境与生产环境中都需要传值，ylbm 只在沙箱环境中需要传值。在乐企平台【测试中心】——【待开通能力测试】——点击待测试能力【查看详情】可以查看能力的ylbm，在乐企平台【测试中心】——【待开通能力测试】——点击待测试能力【查看详情】——【测试详情】界面中会展示该能力下所有接口的 nlbm、fwbm，在沙箱环境和生产环境中nlbm，fwbm 是一致的，而ylbm 只在沙箱环境中使用。 |
| 与乐企服务端通信是否需要加解密验签等步骤？ | 需要。目前乐企采取的安全策略是 ip 白名单+报文对称加密（sm4）。 |
| 企业在月初未申报时下载的授信额度的有效期止为当月申报截止日期，如何理解这句话？ | 可理解为未申报时下载的额度只能用到当月申报截止日期。例如：A 企业在 2022 年 10 月 05 日（尚未申报所属期为 2022 年 9月的增值税）下载了 500 万的授信额度，10 月的申报期结束日是 10月 25 日，该企业已下载的这 500 万的额度只能用到 10 月 25 日，在10 月 25 日以后，如果未申报，即使这 500 万未使用完，也不能继 续使用了；企业申报后，有效期都会延长到月底，企业可以依据新下 载授信额度的有效期更新企业本地未申报前下载的当月授信额度的  有效期，或者调用“调整授信额度有效期”接口更新授信额度有效期。 |
| 基础版开票能力文档中提到的纳税人开票资格判断是需要判断销方纳税人资格？还是购方纳税人资格？或是购销双方纳税人资格都要判断？ | 开票资格即判断开票方的资格；开具销售发票时：开票方即销售方，需要判断销售方纳税人资格；开具收购发票时：开票方即购买方，需要判断购买方纳税人资格。 |
| 能力文档中要求定时更新风险信息？定时更新周期设置多久（天/小时/分钟）？ | 没有作统一要求，试点纳税人可根据企业风控需要，自行决定更新频率，一般情况下可按照“每天”进行设置。 |
| 按文档要求开具发票时需要定时获取更新税收分类编码，定时更新周期设置多久合适？如果没有及时更新使用了无效的税收分类编码会产生什么后果？ | 根据纳税人需求自行决定更新频率，一般情况下可按照“每天”更新的频率进行设置。如果按照无效的税收分类编码开票，可能会导致上传失败的问题，但具体问题还需要根据发票上传返回的具体提示信息进行分析处理；如果上传失败，可及时下载更新税收分类编码。 |

| 企业开票时自行选择税率，是由开票人员主观判断，还是有相应的规则进行判断？ | 按照还权还责于纳税人的总体要求，试点企业需严格按照乐企能力文档中有关数据+规则的要求进行开发，开票时企业开票人员自行选择适用税率，并承担相应的法律责任。 |
| 关于抵扣勾选能力中 2.7 和 2.9 里面提到的“发票异常信息 fpycxx（存在异常发票信息时记录数据）” ， 什么场景下会返回有关的异常发票信息？ | 已勾选在当前税款所属期的发票，在勾选后发生异常（如红冲、作废，异常凭证等）的，在调用申请抵扣统计和申请确认抵扣统计接口时会阻断并返回异常发票信息清单，需要纳税人对相关异常发票撤销勾选后再进行申请抵扣统计和申请确认抵扣统计操作。 |
| 通过乐企下载的未使用的发票号段是否可以跨年使用？ | 不可以。数电发票号码的前 2 位代表数电发票号码的使用年份，不允许跨年度使用。以“22”开头的发票号码为 2022 年度的发票号码，只能在 2022 年使用。企业在 2022 年开具的数电发票，可以在跨年后继续上传（必须确保开票日期中年度在 22 年，不得出现22 开头的票号，开票日期在 23 年），但是在跨年后必须重新下载新一年度的数电发票号码开具数电发票，避免由于发票号码跨年使用造成的发票上传失败。 |
| 当“购买方自然人标志”选择为“否”时，要求【购买方名称】字段长度要大于 4。开票时，如果购买方名称小于等于 4 时是否必须要勾选【自然人】？还是一个提示就可以？ | 答: 系统没有强制要求勾选，结合实际情况由企业自行确定购买方是否为“自然人”。系统会给出提示信息，由企业自行确定和判别。 |

| 出现报文解密失败的情形，是什么原因导致的呢？ | 可能是多种因素导致。企业可从以下几个方面进行排查：一是是否混淆了沙箱密钥和能力密钥，两种密钥有明确的使用范围，沙箱密钥只在沙箱测试时使用，能力密钥用于正式能力开通后使用。二是加密方式是否按照SM4 对称加密。三是更新密钥，使用最新的密钥加密即可（提醒：在能正常使用的情况下，不要随意更新密钥）。 |
| 24.使用单位调用服务时报错“未查询到该能力与对应服务的相关信息”，是什么问题呢？ | 可排查直连单位是否完成沙箱测试。是否给使用单位进行相关能力授权，需要直连单位沙箱测试成功后，能力订阅审批通过，才能将能力授权给使用单位。 |

| 企业于当天 13:20 上传受理成功的一张票,下午 16:05 分查询的时候，还是查不到上传结果，请问如何处理？ | 初步判断原因是发票上传返回的【受理流水号】有效期只有两个小时，发票从上传到结果查询若超出了两个小时，会导致【受理流水号】过期，导致结果查询失败。具体问题需要具体分析，建议企业重新上传下这张发票获取新的【受理流水号】进行结果查询。 |
