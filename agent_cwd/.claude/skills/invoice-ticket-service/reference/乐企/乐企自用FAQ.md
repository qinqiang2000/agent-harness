#### 使用单位登录电子税局没有乐企平台入口
A：办税人员权限问题，请参考：[乐企功能集赋权操作指引.pdf](https://jdpiaozone.yuque.com/attachments/yuque/0/2025/pdf/39256605/1764037823380-e2145e1b-33f5-41d5-9d26-c9a1cea52fb1.pdf)





#### 纳税申报时报错：调用系统服务出错，异常所在 server 名 hxhd dlsw svr005，所属应用：核心征管后端，异常原因：1010010097000045：纳税人未上传发票汇总确认信息
A：企业<font style="color:#DF2A3F;">乐企授权通过</font>后，每月申报增值税前必须将乐企数据进行汇总确认（无论是否有开具发票）

路径：开票管理 -- 抄报税管理 -- 乐企汇总确认

填写组织信息，确定汇总月份。查看是否有本税号的汇总确认信息。

若记录为空，点击新增，详见第4点 汇总确认；若有记录，查看汇总确认是否报错，根据不同报错原因进行不同的处理。



#### 使用单位无授权服务
A：未完成能力授权，请使用单位登录对应报错税号的电子税局，进行能力授权确认，确认完成后重新进行“汇总确认”。

#### 若已做过总分机构汇总确认，总机构和分支机构需保持在统一平台用票。这句话要怎么理解
答：

可以理解为，向税局做过总分机构汇总确认申请的企业，后续就不能再各各为政了，需要总分支机构在一个体系内（即乐企体系内）完成发票的勾选确认等用票动作，以便税局系统核对数据。

 一旦企业办理了“总分机构汇总纳税”的资质，那么总公司和所有分公司在开具和获取发票时，必须使用同一个电子发票服务平台，不能再各自为政。

总分机构汇总确认/汇总纳税：

这是前提。指的是一个企业有总公司（总机构）和分公司（分支机构），经过税务机关的批准，分公司不单独计算并缴纳企业所得税，而是由总公司统一计算，再按一定比例分配给分公司，最后由总公司统一缴纳。

目的： 汇总盈亏，合理降低整体税负。

统一平台：

这里的“平台”通常指的是由税务机关建设或认可的电子发票服务平台。例如，各地的增值税发票综合服务平台、电子税务局内的发票模块等。

“统一”意味着总公司和所有分公司都必须登录并使用同一个平台账号（或通过权限分配在同一个系统内操作），而不是每个机构都有自己的独立平台账号。

用票：

包括所有与发票相关的活动，主要有：

开具发票： 对外开发票。

接收发票： 从供应商那里取得进项发票。

发票勾选认证： 对取得的增值税专用发票进行确认，用于抵扣进项税。

发票查验、入账、归档等。

2. 为什么要这样规定？（背后的逻辑）

这主要是为了满足税务监管和财务核算的要求：

确保汇总纳税数据的准确性：

总公司需要准确掌握所有分公司的收入（开票数据）和成本费用（收票数据），才能正确计算整个企业的应纳税所得额。

如果各机构在不同平台用票，数据是割裂的，总公司无法及时、完整地获取全部分支机构的发票信息，会导致汇总计算错误，产生税务风险。

便于进项税额的统一抵扣：

在汇总纳税模式下，分公司的进项税额通常也由总公司统一抵扣。

如果分公司在另一个平台认证了发票，总公司在自己的平台上看不到，就无法完成抵扣，会造成公司财产的损失。

强化内部控制和风险管理：

统一平台后，总公司可以对分公司的开票权限、开票类型、金额等进行集中管理和监控，防止分公司违规开票，降低企业的整体税务风险。

方便税务机关监管：

对于税务机关来说，他们只需要监控这一个平台的用票情况，就能掌握整个集团（总机构+所有分支机构）的经营全貌，大大提高了征管效率。

3. 举个例子

假设“北京XX科技总公司”在上海和深圳设有分公司，并已办理汇总纳税。

错误做法：

北京总公司使用“北京市电子税务局”的发票平台。

上海分公司使用“上海市电子税务局”的发票平台。

深圳分公司使用“深圳市电子税务局”的发票平台。



结果：三地数据不通，总公司做账和报税时一团乱麻。

正确做法：

经过税务备案，总公司获取一个唯一的平台登录身份。

上海和深圳的分公司作为该总公司的下属单位，在这个统一的平台上被分配了相应的操作权限。

所有分公司的开票、认证等操作，都在总公司的这个统一平台下进行，所有数据实时汇总到总公司层面。

结果：总公司财务部门可以清晰看到全公司的发票流，准确进行财务核算和税务申报。

总结

“若已做过总分机构汇总确认，总机构和分支机构需保持在统一平台用票” 这句话，您可以理解为：

为了实现汇总纳税的管理目标，企业必须像一个“整体”一样来管理发票，将所有分支机构的发票活动都纳入总公司的同一个电子系统里进行，确保数据流的完整、统一和透明。这是企业享受汇总纳税政策红利的同时，必须履行的合规义务。

#### 乐企查验约束
1、发票查验的范围受限。乐企查验仅支持发票购方或销方是本企业的发票，不支持个人抬头发票查验。 其次图片格式的票难度提高，依赖税号识别准确才能查验成功；

2、乐企查验信息不全且性能慢。乐企查验不返回发票状态（红冲/作废等），原税局查验平台支持，乐企需要额外调用乐企用途状态查询接口补偿，该接口为异步接口。整体查验效率偏低，预计等待时间是原来2倍及以上；

建议C端采集对时效性要求高建议保留使用原查验通道，而税局下载时查验目前产品已经是免费了，后续可作为电子税局下载的补偿能力使用。
