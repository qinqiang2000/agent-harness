# <font style="color:rgb(0, 0, 0);">一、资质事宜</font>
## 1.全国性企业的乐企自用申请次序问题
按照《乐企自用直连服务的规范指引》要求，第一步，直连单位（如公司总部）当地主管税务机关申请，申请确认后即可接入开票；第二步，由直连单位向使用单位（如子公司）发送邀请信息；第三步，使用单位确认邀请信息并向使用单位主管税务机关发起使用请求，使用单位主管税务机关确认后，即可接入开票。

## 2. 海外总部企业申请主体问题
按照《乐企自用直连服务的规范指引》要求，乐企自用使用单位与直连单位为同一总分公司、集团企业或具备相互股权控制关系的的企业。可以由一家作为直连单位，其他单位作为使用单位接入时需出具与直连单位的关联关系证明。

# 二、大批量开票问题
## 1.日开票上限问题
目前没有对单户企业数据上传量作出限制，现在可支持单户企业数据上传量达200万。（例如某大型企业一天上传数据量已达到800万）

## 2.百万明细行问题
由于电票平台目前的明细行限制在5000行以内，乐企开具的发票会推送到电票平台，所以不建议明细行过多，否则会导致pdf生成不成功、查询下载明细出错等问题。

# 三、Xml下载问题
乐企停止提供XML下载服务后，纳税人从开票方获得XML。开票方从电票平台开出的数电票，由电票平台生成XML并交付提供下载服务；开票方通过乐企直连模式在自有系统开出数电票的，由开票方自建平台按照税务机关标准生成XML并负责交付。税务机关对接入乐企的直连单位如何生成和交付数电票XML及版式文件有正式的规范，要求乐企接入单位开出数电票后立即通过自有渠道（包括自建APP、电子邮件、其他第三方票夹应用等）完成数电票XML及版式文件的及时交付，税务机关也将对乐企开票方是否按照规范及时生成并交付数电票XML及版式文件进行实时监督。
