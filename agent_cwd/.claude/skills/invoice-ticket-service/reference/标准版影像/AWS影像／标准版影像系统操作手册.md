## 首页
首页：包括影像有误待重扫和影像匹配失败列表

### 影像有误待重扫
功能说明：记录影像系统所有需要重扫的影像单据，用户可以通过点击单据编号，进入到扫描页面，进行扫描

#### 影像单据重扫
功能说明：对需要重扫的单据操作重扫。

单据重扫操作步骤：

1. 影像重扫可以从影像有误待重扫的页面点击单据编号进入扫描页面进行重扫，也可以直接在扫描页面直接扫描，后台判断单据状态是否是重扫的，如果是可直接扫；
2. 重扫可以通过影像扫描页可以通过本地导入的方式进项重扫；
3. 扫描完成点击提交后端自动进行提单流程，提交完成后间隔1~2分钟可以在搜索单据列表查看提交状态

#### 废弃重扫单据
功能说明：对ERP已废弃而影像状态为退回重扫的单据可进行作废，影像单据作废只有管理员或授权的用户才可操作。

步骤：

1. 影像重扫界面勾选需要废弃的影像单据编号；
2. 勾选后点击影像单据作废按钮，弹出二次确认框，点击确认则直接废弃影像单据，废弃成功其关联的发票明细和附件明细都会作相应的删除。

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465569127-7dc74ac6-8e8a-4e81-8a31-c55518ae2c6f.png)

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465569526-2fe372db-4be1-43ef-b461-1dd651a07811.png)

### 影像匹配失败列表
功能说明：记录影像系统所有匹配失败的单据，点击单据编号可以进入到单据匹配结果页，可操作发票数据修改和人工手动匹配；在该界面也可对ERP已废弃但影像状态为匹配失败的影像单据进入作废。

#### 人工匹配
功能说明：对匹配失败或者部分匹配的单据进行人工匹配

操作说明：

1. 匹配失败列表点击需要人工匹配单据的影像编号，进入到匹配结果页
2. 对匹配失败的发票，进行编辑后人工匹配，修改匹配结果完成整单匹配
3. 如果发票影像识别数据有误导致匹配失败，可以对发票数据进行编辑修改，点击编辑在弹出的页面中进行修改；
4. 单据匹配成功后，会自动提交ERP，如果提交失败可以手动点击提交完成提交，也可以在搜索单据列表点击人工重提进行提交；
5. 如果再租户配置页面配置的允许匹配失败的单据提交，可直接点击提交进行提交，页面会给二次确认，确认完成后，提交ERP

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465570037-a70d5d4a-93c6-4f3d-b9a2-777b698dfd84.png)

 

6. 修改完成点击保存修改，完成发票数据的修改，完成修改之后可以进行人工匹配

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465570795-0d9e65ae-4fc0-4861-9ddf-96955f4b6c79.png)

#### 废弃匹配失败单据
功能说明：对ERP已废弃而影像状态为匹配失败的单据可进行影像单据作废，只有管理员或授权的用户才可操作。

步骤：

1. 影像匹配失败界面勾选需要废弃的影像单据编号；
2. 勾选后点击影像单据作废按钮，弹出二次确认框，点击确认则直接废弃影像单据，废弃成功其关联的发票明细和附件明细都会作相应的删除。

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465571363-f6c5cc82-aafc-468d-a749-6cfb5f1e9f09.png)



## 影像扫描
功能说明：扫描员扫描采集单据影像操作界面；

支持票种包括电子普通发票、电子专票、普通发票、专用发票、卷票、定额票、的士票、轮船票、飞机票、客运票、过路过桥票、通用机打发票、通用机打电子发票、通行费电子发票、二手车票、机动车销售发票、其它票、全电专票、全电普票、海关缴款书、火车票、火车票退票、财政电子票据、购置税发票。

使用用户：扫描员

### 安装扫描插件
功能说明:第一次使用影像系统，且需要使用外接扫描仪的用户，需要安装插件，如果没有安装插件不能正常通过扫描仪采集单据影像；

操作步骤：

1. 入口：影像系统-影像扫描-开始扫描
2. 如果没有安装插件会弹出提示框提示安装扫描插件
3. 点击download 下载插件到本地
4. 双击安装包，点击install 应用会自己安装；
5. 安装完成后点击finish完成安装
6. 退出账号，重新进入即可开始扫描采集单据影像

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465571718-c9ab1ce3-dab5-4b21-b8ac-f2f6b7147182.png)

### 采集单据影像
功能说明：扫描员通过影像系统PC端采集单据影像，采集方式有两种，一种是通过外接扫描仪扫描上传，一种是通过本地导入；影像上传成功后可手动添加影像文件标签。

操作步骤：

1. 外接扫描仪扫描
2. 整理好需要采集的单据，放置在扫描仪上，有封面的一面朝下；
3. 点击开始扫描，扫描仪自动扫描单据，并上传影像，影像系统界面上生成对应的影像；
4. 采集完成点击提交，隐形系统后台自动处理影像识别、查验、匹配；
5. 系统流转完成后在可在搜索列表查看最后结果；
6. 支持多笔单据影像一起上传提交



本地导入

1. 在租户配置-<font style="color:rgba(0, 0, 0, 0.85);background-color:rgb(250, 250, 250);">扫描页面配置-本地导入文件类型设置为用户手动分类，</font>点击“导入发票”或“导入附件”的按钮，”导入发票“按钮的入口上传的影像都当发票处理，“导入附件”按钮除了管理员默认显示其它角色默认是不显示的，其它角色需要在角色权限下配置了才会显示。导入附件支持EXCEL、Word、TXT、PPT、heic、多页PDF、OFD和图片格式的文件；设置为系统自动分类，点击“导入影像”或“导入附件”按钮，“导入影像”按钮则根据后台返回的类型所显示相应类型。
2. 选择需要上传的影像本地文件
3. 点击打开，系统自动上传
4. 影像系统生成对应的影像文件
5. 采集完成点击提交；
6. 处理流程和扫描仪上传一致

![](https://cdn.nlark.com/yuque/0/2025/png/38481329/1736839852029-41ec81e1-67fb-497a-95d8-63e7225f71a9.png)

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1725607685218-20f267c3-112a-4757-9489-69a12a28a141.png)

+ 在租户配置-扫描页面配置下设置不支持采集无封面单据，影像单据提交需要有封面，如果没有封面则提交不成功；反之，设置支持则可提交无封面的单据，管理员可在扫描界面通过点击添加单据编号按钮后添加单据编号，非管理员通过授权才可显示添加单据编号按钮。

+ 没有正常识别封面或者没有封面，可以选择其中一张影像点击右侧的转封面，将当前影像转化为封面；
+ 也可以点击新增封面，通过扫描仪或者本地导入的方式新增；

### 补扫影像
功能说明：如果单据影像有缺失，可以操作补扫，往单据中继续添加发票影像

操作说明：

1. 选择需要补扫的单据，点击补扫影像
2. 选择补扫方式：扫描仪或者本地导入；
3. 扫描仪通过扫描添加单据影像，本地导入从本地选择文件导入单据影像；
4. 补扫完成后补扫的影像会添加在所选单据的后面；
5. 点击保存-提交或者影像匹配，完成补扫

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465578692-d2249c5d-b96f-4119-8656-598a0d228a2b.png)

### 替扫影像
功能说明：单据影像有误的时候可以对错误的影像操作替扫，替换掉错误的发票影像。

操作说明：

1. 选择需要替扫的发票影像；
2. 点击替扫影像
3. 选择替扫方式：本地上传或者通过扫描仪扫描导入
4. 保存-提交  完成替扫

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465579247-0c77891f-96bf-43ab-93aa-448a886ff36e.png)

<font style="color:red;">注意：一次只能替扫一张影像</font>

### 移动单据影像
功能说明：单据影像顺序有误，可以通过前移、后移调整顺序，目前只支持单据内影像移动不支持夸单据移动影像；

操作说明：

第一种方式：

1. 选中需要移动的影像，点击前移/后移 调整顺序；
2. 调整完成点击保存-提交完成影像提交；
3. 调阅影像的时候按照我们调整后的顺序展示；

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465580279-856a7957-241f-4dfb-9446-c55985f5ac4c.png)

第二种方式:

影像上传后可能通过拖动左边影像来调整影像位置

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1720608820859-8d4a2fde-47ce-4243-a40a-4ba51f6f46d3.png)



###  添加影像文件标签
功能说明：可对已上传成功的发票或附件添加预设好的文件标签；

操作说明：

1. 选择要添加的影像
2. 点击影像右边的向上的图标按钮
3. 弹出添加文件标签界面，选择需要添加的标签，
4. 选择后点击保存则添加成功
5. 添加成功后会在影像后面显示标签的数量

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1725608217925-896cb363-9dd5-4418-aea5-af0c2bfded8b.png)



## 单据管理
### 单据池
功能说明：记录所有需要采集影像的单据，由业务系统在提单完成后推送单据数据到影像系统系统，做记录，这个列表主要是用于影像系统做签寄管理的时候会用到，做状态判断；

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465581565-4683a57d-c00e-4553-93fe-cde6ac9ee518.png)

### 影像单据列表
功能说明：记录影像系统有影像的单据，所有影像状态的单据都会在这里有记录，根据影像状态的不同，根据单据的状态不同点击单据编号进入的影像页面也不同，还可以在这里操作单据影像的导出，人工重提未提交成功的单据，手工同步收票的数据，导出单据列表的数据。通过详情可查看到当前单据的操作日志。

用户：扫描员

状态说明：

| 单据状态 | 单据说明 | 跳转页面 |
| --- | --- | --- |
| 影像待上传 | 数据传到影像系统，还没有上传影像 | -- |
| 匹配失败 | 影像已上传，但是匹配失败了 | 人工匹配页面 |
| 提交失败 | 影像提交ERP系统失败了 | 调阅界面 |
| 审核中 | 影像提交ERP成功 | 调阅界面 |
| 待重扫 | 影像被退扫 | 扫描界面 |


 

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1720608963932-2058b6e2-095b-4336-92f8-5d124fedbcc0.png)



####  影像调阅
<font style="color:black;">功能说明：影像调阅即影像查看，该界面可查看到当前单据的所有关联的影像；影像调阅界面分为图片模式和列表模式查看，图片模式左边有缩略图显示，而列表模式则是以树状型显示；右边若是发票则显示影像结构化数据，采集渠道和文件标签，而附件则显示文件名称、文件格式、采集渠道和文件标签；支持IE11浏览器/google浏览器/火狐浏览器等调阅。</font>

<font style="color:black;">操作步骤：</font>

1. <font style="color:black;">从ERP单据下点击“查看影像”跳转进入调阅界面；</font>
2. <font style="color:black;">在影像系统-影像单据列表下点击影像单据编码进入调阅界面。</font>

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1725612561309-6c74cef7-2be8-4c45-9d3d-cd2b0052a093.png)

![](https://cdn.nlark.com/yuque/0/2024/png/38481329/1725612584000-a9381199-23bf-43e2-81dd-932441909729.png)

#### 人工重提
功能说明：影像采集完成后点击提交，影像系统后台处理，影像识别、查验、匹配、提交，如果在处理的过程中，流程出现异常，导致没有正常进行下去，可以选中失败的单据点击人工重提，重新推进提交流程

操作说明：

1. 选中：提交失败、查验失败状态的单据（可单笔操作也可以批量操作）；
2. 点击人工重提按钮，重新推动提交流程

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465584625-d7f00ac4-74ce-4e15-b51b-09aa00de3697.png)

 

<font style="color:red;">注意：人工重提点击之后，在没有完成之前不能重复操作，已提交的单据不能重复人工重提</font>

#### 导出单据影像
功能说明：按单导出影像到本地，解压后以单据为单位生成文件夹，点击可查看影像；

操作说明：

1. 勾选需要导出的单据；
2. 点击导出影像
3. 后台下载，目前一次只能下载10个单据，考虑到文件大小的问题；
4. 下载完成，解压压缩包，可查看具体的单据影像

![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465585762-6722d453-ccdb-4b33-a3dd-2f42fbe4d0a4.png)

#### 手工同步

 用户登录影像系统后点击右上角的名称-修改密码进入修改密码界面，输入正确的旧密码和新密码（8-20位大写字母+小写字母+特殊符号+数字）、确认新密码进行修改，修改成功后可用新密码登录。

# 影像模式
## 基础模式
### 模式说明
经办人在业务系统提单完成后，直接在影像系统通过扫描页采集影像，只是采集单据影像，不做影像的识别，采集上传成功通知业务系统影像已上传。支持查验、只识别不查验和不识别查验。

### 适用场景
客户有指定的扫描岗，没有用影像收票，需要在影像系统采集影像

### 模式时序图
![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465614599-365877cb-7d57-411c-a8dc-5b318ac2b2fd.png)

##  匹配模式
### 模式说明
经办人在ERP提单的时候有用收票采集发票，收票保存发票结构化数据和单据的关联关系，标准的发票云收票会通知影像系统，同步发票数据，非标准的影像系统可自己从收票查发票结构化数据，提单人将实物单据递交到扫描岗，扫描岗采集影像，采集完成会调用识别查验的接口对发票影像影像进行是被查验，获取发票结构化数据，将影像的结构化数据和收票的结构化数据进行匹配，匹配成功提交到ERP，匹配失败停留贼影像节点，审核岗在ERP 调阅影像的时候可调阅查看影像。支持查验和只识别不查验。

### 适用场景
有指定的扫描岗，有用收票或者业务系统集成收票功能，可以在提单的时候获取发票结构化数据，需要在影像系统采集影像，需要做系统初审（二次匹配）

### 模式时序图
![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465615691-8e163183-ab44-4a43-83d6-ee455bd87a7c.png)

## 同步模式
### 模式说明
经办人在ERP提单使用收票采集发票源文件，采集完成通知影像影像系统，同步发票图片生成影像

### 适用场景
没有指定的扫描岗，人人可上传影像，不需要在影像系统采集影像

### 模式时序图
![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465616801-b118637c-e4da-461c-90a5-4334ce8a7726.png)

## 稽核模式
### 模式说明
经办人在ERP 提单完成后，同步到影像系统生成影像，审核岗调阅影像进行审核，审核通过后通知提单人通过收单机交实物，交单人在收单机上交实物的时候进行识别匹配，匹配成功后直接归档，流程节点往下流转，匹配失败退回单据，流程节点停留在实物稽核的节点

### 适用场景
企业有使用报销机器人收取实物单据，没有指定的人工扫描岗

### 模式时序图
![](https://cdn.nlark.com/yuque/0/2023/png/29641274/1684465617154-7370e6a3-9027-49c3-9353-73011c414342.png)

## 附件模式
### 模式说明
经办人在提单的时候在业务通过附件助手上传附件影像，导入影像系统生成数据，并通知ERP影像已上传，流程节点往下流转。

### 适用场景
企业内部没有指定的扫描岗，不需要在影像系统采集影像，提单的时候通过提单人自己上传影像，影像包括票据和非发票类文件。
